# Node v123 运行调试说明：managed audit adapter harness runner

本版来源计划：

```text
docs/plans/v122-production-connection-candidate-roadmap.md
```

本版目标：

```text
把 v120 的 managed audit adapter compliance harness
扩展为可选择 runner，同时覆盖 memory 与 file-candidate 两个本地目标。
```

## 运行环境

HTTP smoke 使用安全环境变量：

```text
HOST=127.0.0.1
PORT=4123
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
ACCESS_GUARD_ENFORCEMENT_ENABLED=true
```

HTTP smoke 请求带了 auditor header：

```text
x-orderops-operator-id=auditor-1
x-orderops-roles=auditor
```

截图使用本机 Chrome / Chromium。由于 `npx playwright screenshot` 不支持直接追加自定义 header，本版截图阶段使用 `ACCESS_GUARD_ENFORCEMENT_ENABLED=false` 的 observe-only 模式打开同一 Markdown endpoint；HTTP smoke 已在 enforcement=true 下通过。

没有启动、停止、构建、测试或修改 Java / mini-kv。

## 清理

本版 smoke 启动的 Node 进程已停止。

file-candidate runner 使用的是系统临时目录里的临时 JSONL 文件，服务内部运行完会删除。

临时目录：

```text
.tmp/
```

会在版本收口前清理。
