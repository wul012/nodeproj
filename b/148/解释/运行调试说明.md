## 运行边界

正式 smoke 使用：

```text
ACCESS_GUARD_ENFORCEMENT_ENABLED=true
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

访问受保护接口时使用 operator headers：

```text
x-orderops-operator-id=approver-1
x-orderops-roles=approver,viewer
```

截图时另起临时 Node 进程关闭 access guard，仅用于浏览器截图：

```text
ACCESS_GUARD_ENFORCEMENT_ENABLED=false
```

该截图进程仍保持：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此截图没有触碰 Java / mini-kv。

## 结论

v148 已完成 Node 侧 evidence capture 机制。默认状态会稳定产生 skipped capture；如果后续用户明确打开 Java / mini-kv 只读窗口，同一套 capture 机制可以记录 pass/mixed，再交给 v149 做 production pass evidence verification。
