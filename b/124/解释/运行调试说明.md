# Node v124 运行调试说明：JWKS cache contract rehearsal

本版来源计划：

```text
docs/plans/v122-production-connection-candidate-roadmap.md
```

本版目标：

```text
在 v121 的本地 JWKS fixture 之上增加 cache contract。
覆盖 cache hit、unknown kid、expired cache、rotation marker。
```

## 运行环境

HTTP smoke 使用安全环境变量：

```text
HOST=127.0.0.1
PORT=4124
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
ACCESS_GUARD_ENFORCEMENT_ENABLED=true
ORDEROPS_IDP_ISSUER=https://idp.example
ORDEROPS_IDP_AUDIENCE=orderops-node
ORDEROPS_IDP_JWKS_URL=https://idp.example/.well-known/jwks.json
```

HTTP smoke 请求带了 viewer header：

```text
x-orderops-operator-id=viewer-1
x-orderops-roles=viewer
```

截图使用本机 Chrome / Chromium。由于 `npx playwright screenshot` 不支持直接追加自定义 header，本版截图阶段使用 `ACCESS_GUARD_ENFORCEMENT_ENABLED=false` 的 observe-only 模式打开同一 Markdown endpoint；HTTP smoke 已在 enforcement=true 下通过。

没有启动、停止、构建、测试或修改 Java / mini-kv。
