# v155 运行调试说明：read-only window readiness packet

## 本版目标

v155 把 v152 archive verification、v153 operator runbook、v154 runbook verification 打包成一次真实只读窗口前的人工审阅包。

本版仍然不启动 Java / mini-kv，也不读取另外两个仓库的运行态文件。它只证明：

```text
v152/v153/v154 digest 链完整
真实只读窗口必须由 operator 手动启动 Java / mini-kv
后续窗口必须使用 UPSTREAM_PROBES_ENABLED=true
后续窗口必须保持 UPSTREAM_ACTIONS_ENABLED=false
未启动上游时只能记录 skipped evidence
skipped/mixed evidence 不能变成 production pass
```

## 安全边界

本版没有启动、停止、构建、测试或修改 Java / mini-kv。

Node 临时 smoke / screenshot 服务在验证后停止。

本版继续禁止：

```text
Java replay POST
订单写操作
mini-kv SET / DEL / EXPIRE
UPSTREAM_ACTIONS_ENABLED=true
Node 自动启动 Java / mini-kv
skipped/mixed evidence 冒充 production pass
```
