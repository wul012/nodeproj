# Node v142 运行调试说明

本版目标：

```text
新增 live probe real-read smoke readiness switch。
让真实只读 smoke 的显式开关状态可见，默认保持关闭。
```

## 启动决策

本版只启动 Node 自身做 HTTP smoke 和截图。

没有启动 Java / mini-kv。

原因：

```text
v142 是 readiness switch，不是真实 live smoke；
默认 UPSTREAM_PROBES_ENABLED=false 时，页面应显示 closed-skipped-evidence。
```

## 验证结果

本版验证项：

```text
npm run typecheck
npx vitest run test/productionLiveProbeRealReadSmokeReadinessSwitch.test.ts
npm test
npm run build
HTTP smoke
本机 Chrome 截图
```

测试结果：

```text

```

HTTP smoke 使用安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
ACCESS_GUARD_ENFORCEMENT_ENABLED=true
```

## 调试插曲

第一次 HTTP smoke 的就绪探测没有带 operator header。

在 `ACCESS_GUARD_ENFORCEMENT_ENABLED=true` 下，`/health` 返回 401，被脚本误判为未启动。

修正后：

```text
健康检查带 x-orderops-operator-id / x-orderops-roles
```

Node smoke 正常通过。
