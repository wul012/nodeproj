# v149 运行调试说明

本版本目标：

```text
新增 production pass evidence verification，校验 v148 capture 是否能成为生产 pass 证据。
```

本次只操作 Node 项目：

```text
D:\nodeproj\orderops-node
```

没有启动、构建、测试或修改 Java 高并发项目和 mini-kv。

## 本版本运行结果

正式 HTTP smoke 使用：

```text
ACCESS_GUARD_ENFORCEMENT_ENABLED=true
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

访问受保护接口时使用 operator headers：

```text
x-orderops-operator-id=smoke-operator
x-orderops-roles=viewer,approver
```

smoke 先写入一条 dry-run approval，再读取 v149 接口。结果：

```text
health=ok
profileVersion=production-live-probe-real-read-smoke-production-pass-evidence-verification.v1
verificationState=not-production-pass-evidence
readyForProductionPassEvidenceVerification=false
readyForProductionOperations=false
captureState=captured-skipped
captureMode=skipped
releaseGateDecision=not-production-pass-evidence
productionBlockerCount=0
markdownStatus=200
```

这个结果说明默认不启动 Java / mini-kv 时，v149 能正确把 skipped capture 保留为非生产 pass 证据，而不是误判为 pass。

## 运行边界

v149 不会：

```text
启动 Java
启动 mini-kv
执行 Java replay POST
执行 mini-kv 写命令
打开 UPSTREAM_ACTIONS_ENABLED
把 skipped/mixed capture 冒充 production pass evidence
```

all-pass capture 只有在显式只读窗口打开、五条记录全部通过、release gate 接受时，才会得到：

```text
verificationState=production-pass-evidence-ready
```

即使到这个状态，也仍然保持：

```text
readyForProductionOperations=false
executionAllowed=false
```

## 结论

v149 已完成 Node 侧 production pass evidence verification。默认未启动上游时，它稳定输出 `not-production-pass-evidence`；如果后续显式打开 Java / mini-kv 只读窗口，同一层可以校验 all-pass capture，但仍不会放开生产操作。
