# v154 运行调试说明：operator runbook verification

## 本版目标

v154 消费 v153 operator runbook，复算 `runbookDigest`，并把真实只读联调前清单变成可复核对象。

本版仍然不启动 Java / mini-kv，也不读取另外两个仓库的运行态文件。它只验证 Node 自己生成的 runbook 是否满足：

```text
5 个 operator steps 顺序稳定
Java 目标只包含 GET / contract sample
mini-kv 目标只包含 HEALTH / INFOJSON / STATSJSON / CHECKJSON GET / readonly fixture
forbidden operations 覆盖 Java 写、Java replay POST、mini-kv 写、actions=true、自动启动上游
readyForProductionOperations=false
```

## 安全边界

本版没有启动、停止、构建、测试或修改 Java / mini-kv。

Node 临时 smoke / screenshot 服务在验证后停止。

本版继续禁止：

```text
Java replay POST
订单写操作
mini-kv SET / DEL / EXPIRE
UPSTREAM_ACTIONS_ENABLED=true
Node 自动启动 Java / mini-kv
```
