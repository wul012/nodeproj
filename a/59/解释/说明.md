## 本版目标

在真实执行任何上游动作前，Node 生成统一 preflight evidence bundle，说明当前 intent：

```text
为什么允许继续 dry-run
为什么必须阻断
有哪些 warning 需要人工看过
收集到了哪些 Java / mini-kv 只读证据
```

本版仍然只做 Node 控制面能力，不修改 Java 项目，不修改 mini-kv 项目。

## 本版实现

- `src/services/operationPreflight.ts`
  - 新增 `OperationPreflightService`
  - 汇总 intent summary、action plan、policy、confirmation、rate limit、安全开关、dispatch history
  - 按 action / target 收集 Java replay readiness、mini-kv COMMANDSJSON、mini-kv KEYSJSON evidence
  - 输出 `hardBlockers`、`warnings`、`recommendedNextActions`、`readyForDryRunDispatch`

- `src/routes/operationPreflightRoutes.ts`
  - 新增 `GET /api/v1/operation-intents/:intentId/preflight`
  - 支持 `failedEventId` 和 `keyPrefix` 查询参数
  - 查询参数使用 schema 限制安全字符

- `src/app.ts`
  - 注册 preflight route
  - 放在 operation intent 通用路由之前，避免 `:intentId` 路由抢占子路径

- `src/ui/dashboard.ts`
  - Intent 操作区新增 `Preflight` 按钮
  - 点击后带上当前 `failedEventId` / `kvPrefix` 请求 preflight JSON

- `test/operationPreflight.test.ts`
  - 覆盖默认安全阻断
  - 覆盖 Java replay readiness blocker
  - 覆盖 mini-kv command catalog + key inventory evidence
  - 覆盖缺少 `failedEventId` 的上下文阻断
  - 覆盖 Fastify route smoke

- `docs/plans/v59-post-preflight-control-roadmap.md`
  - 新增由 Node v59 衍生的下一阶段计划
  - 收敛为 Node v60、Java v39、mini-kv v48、Node v61、Node v62

### 04 默认安全 smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
PORT=4599
```

流程：

```text
POST /api/v1/operation-intents       action=kv-set
GET  /api/v1/operation-intents/:id/preflight?keyPrefix=orderops:
```

结果：

```text
intentStatus=blocked
hardBlockers=UPSTREAM_ACTIONS_ENABLED=false,INTENT_STATUS_BLOCKED
warnings=WRITE_RISK_REQUIRES_EXPLICIT_REVIEW,UPSTREAM_PROBES_SKIPPED
commandEvidence=skipped
keyEvidence=skipped
```

意义：默认安全环境不会误触碰 mini-kv probe 或上游写动作。

### 05 mock mini-kv preflight smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=true
MINIKV_HOST=127.0.0.1
MINIKV_PORT=16402
PORT=4602
```

本轮只启动本地 Node TCP mock 模拟 mini-kv v47，不启动真实 mini-kv，不启动 Java。

流程：

```text
POST /api/v1/operation-intents       action=kv-set
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches    mode=dry-run
GET  /api/v1/operation-intents/:id/preflight?keyPrefix=orderops:
```

结果：

```text
dispatchStatus=dry-run-completed
readyForDryRunDispatch=true
miniKvCommandCatalog=available
miniKvKeyInventory=available
warnings=WRITE_RISK_REQUIRES_EXPLICIT_REVIEW,MINIKV_MUTATING_COMMAND
```

意义：Node v59 能把 confirmed intent、dry-run dispatch、mini-kv command risk 和 key inventory 汇总成统一 preflight evidence bundle。
