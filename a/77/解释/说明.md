# Node v77：Fixture evidence archive snapshot

## 来源

本版依照 `docs/plans/v75-post-fixture-diagnostics-roadmap.md` 推进，由 Node v76 的 Dashboard fixture diagnostics panel 衍生。

v76 已经把 fixture report 和 drift diagnostics 放进 Dashboard。v77 在此基础上新增只读 archive snapshot，用于证明当前 Node smoke 使用的是哪一版 Java / mini-kv fixture 证据。

## 本版完成

- 新增 archive snapshot 服务：

```text
src/services/upstreamContractFixtureArchive.ts
```

- 新增接口：

```text
GET /api/v1/upstream-contract-fixtures/archive-snapshot
GET /api/v1/upstream-contract-fixtures/archive-snapshot?format=markdown
```

- snapshot 包含：

```text
fixtureArchiveDigest
fixtureReportDigest
driftDigest
Java execution-contract fixture path
mini-kv CHECKJSON fixture path
Java contractDigest / approvalDigest / replayEligibilityDigest
mini-kv commandDigest / readOnly / executionAllowed / sideEffectCount
drift issueCount / errorCount / missingMappingCount
evidence endpoint 列表
```

## 本版不做

- 不做数据库持久化
- 不替代 execution gate archive
- 不把 fixture archive 当成真实执行许可
- 不自动修复 fixture drift
- 不修改 Java / mini-kv
- 不调用 Java replay POST
- 不执行 mini-kv 写命令

## 收口判断

v77 把 fixture report 与 drift diagnostics 固化成一份可归档 snapshot。后续 Node v78 做多场景矩阵时，可以继续沿用这种模式，把每个 fixture 场景的 digest、来源路径和关键字段都放进可审计证据中。
