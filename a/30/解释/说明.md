# OrderOps Node V30 运行调试归档

## 本版目标

V30 新增本地 promotion handoff package verification：

- `GET /api/v1/ops/promotion-archive/handoff-package/verification` 输出 JSON verification。
- `GET /api/v1/ops/promotion-archive/handoff-package/verification?format=markdown` 输出 Markdown verification report。
- 复算 V29 `packageDigest`，确认交接包索引自洽。
- 逐个复核五个 attachment 的 `valid`、`source` 和 `digest`。
- 对比 package name、archive name、state、handoffReady、sealDigest、manifestDigest、verificationDigest、summary 和 nextActions。
- Dashboard 新增 `Package Verification` 和 `Package Verification Report` 入口。
- 本功能只读取 Node 本地内存状态，不触碰 Java / mini-kv。

### 9. 逐个复核 attachments

文件：`src/services/opsPromotionArchiveBundle.ts`

```ts
const attachmentChecks = input.handoffPackage.attachments.map((attachment) => {
  const expected = expectedPackage.attachments.find((candidate) => candidate.name === attachment.name);
  const validMatches = expected?.valid === attachment.valid;
  const sourceMatches = expected?.source === attachment.source;
  const expectedDigest = expected?.digest ?? { algorithm: "sha256" as const, value: digestStable({ missing: attachment.name }) };
  const digestMatches = attachment.digest.value === expectedDigest.value;
```

```ts
return {
  name: attachment.name,
  valid: expected !== undefined && validMatches && sourceMatches && digestMatches,
  validMatches,
  sourceMatches,
  digestMatches,
  packageDigest: { ...attachment.digest },
  recomputedDigest: expectedDigest,
  source: attachment.source,
};
```
