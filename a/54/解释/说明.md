# 第五十四版运行调试归档说明

本轮版本目标：

```text
Node v54 接入 mini-kv v45 INFOJSON，让 /api/v1/upstreams/overview 能识别 mini-kv 的版本、协议、uptime、WAL 和 metrics 模式。
```

本轮只做 Node 侧只读接入，不修改 Java，不修改 mini-kv 源码，不开放上游写动作。

---

# 1. 代码改动

## `src/clients/miniKvClient.ts`

新增：

- `MiniKvInfoJson`
- `MiniKvInfoJsonResult`
- `MiniKvClient.infoJson()`
- `parseMiniKvInfoJson()`

并把 raw command 白名单补上 `INFOJSON`。

核心意义：

```text
Node 可以把 mini-kv v45 的 INFOJSON 文本响应解析成稳定对象，而不是只把它当作普通字符串。
```

## `src/services/opsSnapshotService.ts`

新增 `MiniKvInfoJsonProbe`，并让 mini-kv 探测同时读取：

```text
PING
HEALTH
STATSJSON
INFOJSON
```

如果 `PING/HEALTH/STATSJSON` 正常但 `INFOJSON` 不可用，mini-kv 会进入 `degraded`，不会误判成 `offline`。

## `src/services/upstreamOverview.ts`

`/api/v1/upstreams/overview` 的 mini-kv signals 新增：

```text
infoJsonAvailable
infoJsonLatencyMs
version
protocol
uptimeSeconds
maxRequestBytes
metricsEnabled
```

并把 `INFOJSON` 加入 mini-kv 的 `readSignals`。

# 5. 真实 mini-kv v45 联调 smoke

第一次直接使用：

```text
D:\C\mini-kv\cmake-build-debug\minikv_server.exe
```

发现它仍是旧构建，`INFOJSON` 返回：

```text
ERR unknown command
```

因此使用独立临时目录构建 mini-kv 当前 v45 源码：

```powershell
$cmake = 'D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe'
$env:PATH = 'D:\CLION\CLion 2025.1.1\bin\mingw\bin;' + $env:PATH
& $cmake -S . -B build-node-v54-smoke -G 'MinGW Makefiles' -DCMAKE_BUILD_TYPE=Debug
& $cmake --build build-node-v54-smoke --parallel 2
```

构建结果：

```text
Exit code: 0
100% Built target minikv_metrics_file_tests
```

随后启动临时 mini-kv v45 server + Node，`/api/v1/upstreams/overview` 返回关键摘要：

```json
{
  "nodeHealth": "ok",
  "overallState": "offline",
  "javaState": "offline",
  "miniKvState": "online",
  "miniKvVersion": "0.45.0",
  "miniKvProtocol": "inline,resp",
  "miniKvUptimeSeconds": 0,
  "miniKvLiveKeys": 0,
  "miniKvWalEnabled": false,
  "miniKvMetricsEnabled": true,
  "upstreamActionsEnabled": false
}
```

说明：

```text
overallState=offline 是因为本轮只启动 mini-kv，没有启动 Java。
miniKvState=online 证明 Node v54 已能读取 mini-kv v45 INFOJSON。
upstreamActionsEnabled=false 证明真实写动作仍保持关闭。
```
