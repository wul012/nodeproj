# Node v65 说明：接入 Java v40 与 mini-kv v49 上游审批证据

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v65：统一接入 Java v40 与 mini-kv v49 的新增上游证据
```

本版完成后已新增后续计划：

```text
docs/plans/v65-post-upstream-evidence-roadmap.md
```

该计划说明：由 Node v65 衍生，后续先推进 Node v66，再让 Java v41 + mini-kv v50 可并行独立推进。

## 本版目标

把 v64 的本地 approval evidence report 升级为：

```text
request + decision + digest chain
+ Java v40 failed-event approval-status
+ mini-kv v49 EXPLAINJSON side_effects
+ upstreamEvidence verification
```

仍然只做只读证据接入，不做真实执行。

## 本版实现

- `src/clients/orderPlatformClient.ts`
  - 新增 `OrderPlatformFailedEventApprovalStatus`
  - 新增 `failedEventApprovalStatus(failedEventId)`

- `src/routes/orderPlatformRoutes.ts`
  - 新增 `GET /api/v1/order-platform/failed-events/:failedEventId/approval-status`
  - 继续由 `UPSTREAM_PROBES_ENABLED` 保护

- `src/clients/miniKvClient.ts`
  - `MiniKvExplainJson` 新增 `side_effects?: string[]`
  - parser 校验 `side_effects` 必须是字符串数组

- `src/services/operationApprovalEvidence.ts`
  - 新增 `OperationApprovalEvidenceService`
  - evidence report 新增 `upstreamEvidence`
  - summary 新增 `javaApprovalStatus`、`javaApprovedForReplay`、`miniKvExplainCoverage`、`miniKvSideEffects`
  - evidence digest 纳入 `upstreamEvidence`
  - verification 新增 `upstreamEvidenceMatchesSummary`
  - Markdown 新增 `Upstream Evidence` 与 `mini-kv side_effects` 段落

- `src/routes/operationApprovalEvidenceRoutes.ts`
  - evidence / verification 改为通过 `OperationApprovalEvidenceService` 生成报告

- `src/app.ts`
  - 给 evidence routes 注入 `config`、`orderPlatform`、`miniKv`

- `test/operationApprovalEvidence.test.ts`
  - 覆盖 Java v40 approval-status evidence
  - 覆盖 mini-kv v49 side_effects evidence
  - 覆盖 verification 的 `upstreamEvidenceMatchesSummary=true`

## 本版不做

```text
不调用 Java replay / approval POST
不执行 mini-kv SET / DEL / EXPIRE
不修改 Java 项目
不修改 mini-kv 项目
不把上游缺失证据自动判定为可执行
```
