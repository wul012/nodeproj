## 本版目标

Node 接入 Java v38：

```text
GET /api/v1/failed-events/{id}/replay-readiness
```

Node 对外新增：

```text
GET /api/v1/order-platform/failed-events/:failedEventId/replay-readiness
```

并在 Dashboard 增加 failed event id 查询入口。

## 本版实现

- `src/clients/orderPlatformClient.ts`
  - 新增 `OrderPlatformFailedEventReplayReadiness`
  - 新增 `failedEventReplayReadiness(failedEventId)`

- `src/services/upstreamActionGuard.ts`
  - 新增 `assertUpstreamProbesEnabled`
  - 默认 `UPSTREAM_PROBES_ENABLED=false` 时 readiness 路由返回 `UPSTREAM_PROBES_DISABLED`

- `src/routes/orderPlatformRoutes.ts`
  - 新增 Node readiness 路由
  - `failedEventId` 只接受数字字符串
  - 只读调用走 probe gate，不走 action gate

- `src/services/actionPlanner.ts`
  - 新增 `failed-event-replay-readiness` action
  - 新增 `ActionGate = probes | actions`
  - readiness action 使用 `UPSTREAM_PROBES_ENABLED`

- `src/services/operationIntent.ts`
  - policy blocker 增加 `UPSTREAM_PROBES_ENABLED=false`

- `src/ui/dashboard.ts`
  - Order Platform 卡片新增 Replay Readiness 查询入口
  - 显示 readiness、blockers、next actions

## 本版不做

不创建 Java approval

### 04 安全 smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
PORT=4197
```

访问：

```text
GET /api/v1/order-platform/failed-events/42/replay-readiness
```

结果：

```text
HTTP 403
error=UPSTREAM_PROBES_DISABLED
```

意义：默认安全环境不会误触碰 Java readiness endpoint。

### 05 mock Java v38 只读 smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=false
ORDER_PLATFORM_URL=http://127.0.0.1:18084
PORT=4197
```

本轮只启动本地 Node mock 作为 Java v38 readiness 响应，不启动真实 Java / mini-kv。

结果：

```text
eligible=false
approval=true
blockers=REPLAY_APPROVAL_PENDING
next=REVIEW_REPLAY_APPROVAL
```

意义：Node v57 能只读代理 Java v38 replay readiness，并保留 action gate 关闭状态。
