# Node v63 说明：Approval decision record

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v63：Approval decision record
```

本版接在：

```text
Node v62：Operation approval request ledger
```

目标是给本地 approval request 增加 reviewer approve / reject 决策记录。

## 本版目标

新增本地 operation approval decision ledger：

```text
decisionId
requestId
intentId
previewDigest
decision=approved|rejected
reviewer
reason
createdAt
decisionDigest
upstreamTouched=false
```

本版仍然不执行真实 Java replay，不执行真实 mini-kv 写命令，只记录审批决定。

## 本版实现

- `src/services/operationApprovalRequest.ts`
  - 新增 `recordDecision`
  - 只允许 `pending` approval request 被决定
  - reviewer 决策后更新 request status / reviewer / decisionReason / updatedAt

- `src/services/operationApprovalDecision.ts`
  - 新增 `OperationApprovalDecisionLedger`
  - 新增 `digestOperationApprovalDecision`
  - 新增 `renderOperationApprovalDecisionMarkdown`
  - 同一个 request 只允许记录一次 decision
  - decision 记录固定 `upstreamTouched=false`

- `src/routes/operationApprovalDecisionRoutes.ts`
  - 新增 `POST /api/v1/operation-approval-requests/:requestId/decision`
  - 新增 `GET /api/v1/operation-approval-decisions`
  - 新增 `GET /api/v1/operation-approval-decisions/:decisionId`
  - 新增 `?format=markdown`

- `src/app.ts`
  - 注入 `OperationApprovalDecisionLedger`
  - 注册 decision routes

- `src/ui/dashboard.ts`
  - 新增 approval request id 输入
  - 新增 decision reason 输入
  - 新增 `Approve Request`
  - 新增 `Reject Request`
  - 新增 `Approval Decisions`

- `test/operationApprovalDecision.test.ts`
  - 覆盖 blocked request 不能被 approve
  - 覆盖 pending request 可以被 approve
  - 覆盖 request status 被更新
  - 覆盖 decision digest 复算
  - 覆盖 Markdown 输出
  - 覆盖重复 decision 被拒绝

### 04 默认安全 decision smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

流程：

```text
POST /api/v1/operation-intents                         action=kv-set
POST /api/v1/operation-approval-requests              key=orderops:preview,value=preview-value
POST /api/v1/operation-approval-requests/:id/decision decision=approved
```

结果：

```text
approvalStatus=rejected
decisionStatusCode=409
decisionError=APPROVAL_REQUEST_NOT_DECIDABLE
```

意义：默认安全环境产生的 blocked/rejected approval request 不能被 reviewer 批准。

### 05 mock mini-kv approval decision smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=true
MINIKV_HOST=127.0.0.1
MINIKV_PORT=<本地 mock TCP 端口>
```

本轮只启动本地 mock mini-kv TCP 服务，不启动真实 mini-kv，不启动真实 Java。

流程：

```text
POST /api/v1/operation-intents                         action=kv-set
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches                      mode=dry-run
POST /api/v1/operation-approval-requests              keyPrefix=orderops:,key=orderops:preview,value=preview-value
POST /api/v1/operation-approval-requests/:id/decision decision=approved
GET  /api/v1/operation-approval-decisions/:decisionId?format=markdown
```

结果：

```text
requestStatusBeforeDecision=pending
requestStatusAfterDecision=approved
decision=approved
upstreamTouched=false
decisionDigest=sha256:<64 hex>
```

mock mini-kv 收到：

```text
COMMANDSJSON
KEYSJSON orderops:
EXPLAINJSON SET orderops:preview preview-value
```
