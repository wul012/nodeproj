# v39 说明：Promotion Release Evidence

## 本版目标

v39 在 v38 的 `handoff completion verification` 后面增加最终 release evidence：

- 新增 `GET /api/v1/ops/promotion-archive/release-evidence`
- 新增 `GET /api/v1/ops/promotion-archive/release-evidence?format=markdown`
- Dashboard 新增 `Release Evidence` 和 `Release Evidence Report`
- 代码讲解新增 `代码讲解记录/43-ops-promotion-release-evidence-v39.md`

## 核心改动

`src/services/opsPromotionArchiveBundle.ts` 新增：

- `OpsPromotionReleaseEvidence`
- `OpsPromotionReleaseEvidenceItem`
- `createOpsPromotionReleaseEvidence`
- `renderOpsPromotionReleaseEvidenceMarkdown`

release evidence 汇总：

- handoff completion
- verified handoff completion
- handoff closure
- verified handoff closure
- final closeout state

## 运行调试结果

本轮本地验证命令：

```powershell
npm run typecheck
npm test
npm run build
```

结果：

- `typecheck` 通过
- `vitest` 通过：18 个测试文件，79 条测试
- `build` 通过

HTTP smoke 使用本项目 Node dist 服务：

- host: `127.0.0.1`
- port: `4100`
- pid: `14292`
- `UPSTREAM_PROBES_ENABLED=false`
- `UPSTREAM_ACTIONS_ENABLED=false`

Smoke 覆盖：

- `/health`
- `/api/v1/ops/promotion-archive/release-evidence`
- `POST /api/v1/ops/promotion-decisions`
- `/api/v1/ops/promotion-archive/handoff-completion/verification`
- `/api/v1/ops/promotion-archive/release-evidence?format=markdown`
- `/`

结果：

- health 为 `ok`
- 空账本 release evidence：`valid=true`，`state=not-started`，`handoffReady=false`
- blocked 决策后 release evidence：`valid=true`，`state=blocked`，`handoffReady=false`
- evidence digest 为 64 位 SHA-256
- completion digest 等于 verified completion digest
- closure digest 等于 verified closure digest
- 5 个 evidence item 均 `valid=true`
- Markdown 包含 `# Promotion release evidence`
- Dashboard 包含两个新增按钮和新增 endpoint

## 自动化

本轮还创建了线程心跳自动化 `orderops`：

- 默认每天继续推进下一个版本开发运行调试
- 保持不打断 Java / mini-kv
- 完成后自动验证、归档、提交、打标签并推送

## 对 Java / mini-kv 的影响

本轮没有启动、停止、构建、测试或修改 Java 高并发项目和 mini-kv。

Node smoke 使用了安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此不会主动探测或调用 Java / mini-kv 上游。
