# OrderOps Node V9 运行调试归档

## 本版目标

V9 新增本地 dry-run dispatch ledger：

- 只有 confirmed intent 才能进入 dry-run dispatch completed。
- blocked / pending / expired intent dispatch 会返回 rejected。
- 每次 dispatch 都保存在 Node 内存 ledger 中。
- dispatch 会写入 intent timeline。
- dispatch 永远返回 `upstreamTouched=false`，不会调用 Java / mini-kv。

### 3. 非 confirmed intent 会被拒绝

文件：`src/services/operationDispatch.ts`

```ts
if (intent.status !== "confirmed") {
  const dispatch: OperationDispatch = {
    ...base,
    status: "rejected",
    rejection: {
      code: "INTENT_NOT_CONFIRMED",
      message: "Only confirmed operation intents can be dispatched",
      intentStatus: intent.status,

```

### 5. IntentStore 开放 dispatch 事件入口

文件：`src/services/operationIntent.ts`

```ts
recordDispatchEvent(
  intentId: string,
  type: Extract<OperationIntentEventType, "intent.dispatch.rejected" | "intent.dispatch.dry_run_completed">,
  message: string,
  details?: Record<string, unknown>,
): void {
  const intent = this.requireIntent(intentId);
  this.recordEvent(intent, type, message, details);
}
```
