# v37 说明：Promotion Handoff Completion

## 本版目标

v37 在 v36 的 `handoff closure verification` 后面增加最终完成记录：

- 新增 `GET /api/v1/ops/promotion-archive/handoff-completion`
- 新增 `GET /api/v1/ops/promotion-archive/handoff-completion?format=markdown`
- Dashboard 新增 `Handoff Completion` 和 `Completion Report`
- 代码讲解新增 `代码讲解记录/41-ops-promotion-handoff-completion-v37.md`

## 核心改动

`src/services/opsPromotionArchiveBundle.ts` 新增：

- `OpsPromotionHandoffCompletion`
- `OpsPromotionHandoffCompletionStep`
- `createOpsPromotionHandoffCompletion`
- `renderOpsPromotionHandoffCompletionMarkdown`

completion record 汇总：

- closure digest
- verified closure digest
- decision 摘要
- closure verification 摘要
- 5 个 completion step
- completion digest

5 个 completion step：

```text
closure-created
closure-verified
closure-items-verified
receipt-chain-verified
handoff-readiness-recorded
```

其中 `valid` 表示证据链是否自洽，`ready` 表示是否真正达到最终交接条件。

- `UPSTREAM_ACTIONS_ENABLED=false`

Smoke 覆盖：

- `/health`

- readiness step 在 blocked/empty 下 `ready=false`
- Markdown 包含 `# Promotion handoff completion`

## 对 Java / mini-kv 的影响

本轮没有启动、停止、构建、测试或修改 Java 高并发项目和 mini-kv。

Node smoke 使用了安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此不会主动探测或调用 Java / mini-kv 上游。
