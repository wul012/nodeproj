# Node v66 说明：Approval evidence handoff bundle

## 来源计划

本版按最新计划执行：

```text
docs/plans/v65-post-upstream-evidence-roadmap.md
```

对应版本：

```text
Node v66：Approval evidence handoff bundle
```

## 本版目标

把 v65 的审批证据整理成可交付交接包：

```text
approval request
approval decision
approval evidence report
approval evidence verification
upstream evidence summary
artifact digest summary
bundle digest
```

本版仍然只做证据归档，不做真实执行。

## 本版实现

- `src/services/operationApprovalHandoffBundle.ts`
  - 新增 `createOperationApprovalHandoffBundle`
  - 新增 `renderOperationApprovalHandoffBundleMarkdown`
  - 新增 `bundleDigest`
  - 新增五类 artifact 摘要：
    - `approval-request`
    - `approval-decision`
    - `approval-evidence-report`
    - `approval-evidence-verification`
    - `upstream-evidence`

- `src/routes/operationApprovalEvidenceRoutes.ts`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/evidence-bundle`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/evidence-bundle?format=markdown`
  - 同一次请求内使用同一份 evidence report 生成 verification 和 bundle

- `test/operationApprovalEvidence.test.ts`
  - 覆盖 handoff bundle JSON
  - 覆盖 handoff bundle Markdown
  - 覆盖 `handoffReady=true`
  - 覆盖五类 artifact 均 `present=true` 且 `valid=true`

- `docs/plans/v65-post-upstream-evidence-roadmap.md`
  - 补充 Node v66 实施收口
  - 明确下一步为 `Java v41 + mini-kv v50` 可并行独立推进

- `代码讲解记录/70-operation-approval-handoff-bundle-v66.md`
  - 新增本版代码讲解

## 本版不做

```text
不新增真实执行入口
不调用 Java replay / approval POST
不执行 mini-kv SET / DEL / EXPIRE
不把 bundle 写入数据库
不修改 Java 项目
不修改 mini-kv 项目
```
