# Node v69：execution gate preview archive record

## 来源

本版依照 `docs/plans/v68-post-execution-gate-roadmap.md` 推进，目标是把 v68 的 execution gate preview 固化成本地 archive record。

## 本版完成

- 新增 `OperationApprovalExecutionGateArchiveLedger`
- 新增 execution gate archive record：
  - `archiveId`
  - `sequence`
  - `createdAt`
  - `reviewer`
  - `reviewerNote`
  - `gateDigest`
  - `bundleDigest`
  - `archiveDigest`
  - preview 快照
- 新增 endpoint：
  - `POST /api/v1/operation-approval-requests/:requestId/execution-gate-archive`
  - `GET /api/v1/operation-approval-execution-gate-archives`
  - `GET /api/v1/operation-approval-execution-gate-archives/:archiveId`
  - `GET /api/v1/operation-approval-execution-gate-archives/:archiveId?format=markdown`

## 本版不做

- 不调用 Java replay POST
- 不执行 mini-kv SET/DEL/EXPIRE
- 不把 archive record 持久化到数据库
- 不修改 Java 或 mini-kv 项目

## 收口判断

v69 已把 execution gate preview 从“即时预览”推进到“本地可归档记录”。下一版 v70 应复核 archiveDigest、gateDigest、bundleDigest、requestId、decisionId 与当前证据链是否一致。
