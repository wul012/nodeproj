# Node v62 说明：Operation approval request ledger

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v62：Operation approval request ledger
```

本版由 Node v52 统一观察台主线继续衍生，接在：

```text
Node v60：Preflight Markdown report + digest
Node v61：Operation execution preview
```

## 本版目标

新增本地 operation approval request ledger：

```text
preflightDigest
previewDigest
requestedBy / reviewer
status=pending|rejected|expired
decisionReason
Markdown report
Dashboard request/list 入口
```

本版仍然不执行真实 Java replay，不执行真实 mini-kv 写命令，只把预检和预演结果绑定成本地审批请求记录。

## 本版实现

- `src/services/operationApprovalRequest.ts`
  - 新增 `OperationApprovalRequestLedger`
  - 新增 `digestOperationExecutionPreview`
  - 新增 `renderOperationApprovalRequestMarkdown`
  - 根据 `readyForApprovalRequest` 生成 `pending` 或 `rejected`
  - pending 请求默认 10 分钟过期为 `expired`

- `src/routes/operationApprovalRequestRoutes.ts`
  - 新增 `POST /api/v1/operation-approval-requests`
  - 新增 `GET /api/v1/operation-approval-requests`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId`
  - 新增 `?format=markdown`

- `src/app.ts`
  - 注入 `OperationApprovalRequestLedger`
  - 注册 approval request routes

- `src/ui/dashboard.ts`
  - Intent 操作区新增 `Request Approval`
  - Intent 操作区新增 `Approval Requests`

- `test/operationApprovalRequest.test.ts`
  - 覆盖默认安全 gate 下 rejected 请求
  - 覆盖 mock mini-kv 下 pending 请求
  - 覆盖 Markdown 查询和 preview digest 复算

- `docs/plans/v59-post-preflight-control-roadmap.md`
  - 续写由 v62 衍生的 Node v63/v64、Java v40、mini-kv v49 方向

### 04 默认安全 approval request smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

流程：

```text
POST /api/v1/operation-intents              action=kv-set
POST /api/v1/operation-approval-requests   key=orderops:preview,value=preview-value
```

结果：

```text
status=rejected
previewState=blocked
readyForApprovalRequest=false
hardBlockers=UPSTREAM_ACTIONS_ENABLED=false,INTENT_STATUS_BLOCKED
miniKvExplain=skipped
```

意义：默认安全环境不会触碰 Java 或 mini-kv。

### 05 mock mini-kv approval request smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=true
MINIKV_HOST=127.0.0.1
MINIKV_PORT=<本地 mock TCP 端口>
```

本轮只启动本地 mock mini-kv TCP 服务，不启动真实 mini-kv，不启动真实 Java。

流程：

```text
POST /api/v1/operation-intents              action=kv-set
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches           mode=dry-run
POST /api/v1/operation-approval-requests   keyPrefix=orderops:,key=orderops:preview,value=preview-value
GET  /api/v1/operation-approval-requests/:requestId?format=markdown
```

结果：

```text
status=pending
previewState=review-required
readyForApprovalRequest=true
expectedSideEffects=mini-kv.store-would-mutate,mini-kv.wal-would-be-touched
hardBlockers=none
```

mock mini-kv 收到：

```text
COMMANDSJSON
KEYSJSON orderops:
EXPLAINJSON SET orderops:preview preview-value
```
