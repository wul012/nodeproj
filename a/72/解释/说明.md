# Node v72：execution-contract archive bundle

## 来源

本版依照 `docs/plans/v71-post-execution-contract-roadmap.md` 推进，由 Node v71 的 execution-contract evidence 衍生。

v71 已完成：

```text
Java replay-execution-contract evidence
mini-kv CHECKJSON execution contract evidence
execution gate preview contract checks
```

v72 在这个基础上把执行契约证据整理成单独的 archive bundle。

## 本版完成

- 新增 `OperationApprovalExecutionContractArchiveBundle`
- 新增接口：

```text
GET /api/v1/operation-approval-execution-gate-archives/:archiveId/execution-contract-bundle
GET /api/v1/operation-approval-execution-gate-archives/:archiveId/execution-contract-bundle?format=markdown
```

- bundle 重点串联：

```text
archiveDigest
gateDigest
handoffBundleDigest
archiveVerificationDigest
Java contractDigest
mini-kv CHECKJSON contract snapshot
```

- Java 场景可展示：

```text
javaExecutionContractStatus
javaContractVersion
javaContractDigest
javaReplayPreconditionsSatisfied
javaDigestVerificationMode
```

- mini-kv 场景可展示：

```text
miniKvExecutionContractStatus
miniKvCommandDigest
miniKvCheckReadOnly
miniKvCheckExecutionAllowed
miniKvCheckDurability
```

## 本版不做

- 不新增真实 execution endpoint
- 不调用 Java replay POST
- 不执行 mini-kv SET / DEL / EXPIRE
- 不修改 Java 或 mini-kv
- 不把 bundle 持久化到数据库

## 收口判断

v72 把 v71 的执行契约证据升级成可单独归档的 bundle。现在 Node 能把 execution gate archive、gate preview、handoff bundle、archive verification、Java contractDigest、mini-kv CHECKJSON contract 组织成一条可引用证据链，但仍然保持 preview-only，不触碰真实上游写操作。
