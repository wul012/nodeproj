# Node v70：execution gate archive verification

## 来源

本版依照 `docs/plans/v68-post-execution-gate-roadmap.md` 推进，目标是复核 v69 execution gate archive record 的关键 digest 与 request/decision 链路是否一致。

## 本版完成

- 新增 execution gate archive verification
- 复核内容包括：
  - `archiveDigest`
  - `gateDigest`
  - `bundleDigest`
  - `requestId`
  - `decisionId`
  - `intentId`
  - 当前 request ledger 对齐
  - 当前 decision ledger 对齐
  - `executionAllowed=false`
  - `previewOnly=true`
- 新增 endpoint：
  - `GET /api/v1/operation-approval-execution-gate-archives/:archiveId/verification`
  - `GET /api/v1/operation-approval-execution-gate-archives/:archiveId/verification?format=markdown`

## 本版不做

- 不新增真实 execution endpoint
- 不调用 Java replay POST
- 不执行 mini-kv SET/DEL/EXPIRE
- 不修改 Java 或 mini-kv 项目
- 不把 verification mismatch 自动修复为通过

## 收口判断

v70 已把 v69 archive record 从“可归档”推进到“可复核”。下一步按计划不应继续 Node v71，除非 Java v42 + mini-kv v51 的只读 execution-contract / CHECKJSON 能力已经完成，或用户明确要求 Node 继续做本地占位能力。
