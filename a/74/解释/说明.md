# Node v74：fixture-driven smoke

## 来源

本版依照 `docs/plans/v71-post-execution-contract-roadmap.md` 推进。

前置版本已经完成：

```text
Java v43：replay-execution-contract 稳定 sample
mini-kv v52：CHECKJSON 稳定 fixture sample
```

v74 的目标是让 Node 不再只依赖手写 mock 字段，而是先读取上游稳定样本，再用这些样本驱动本地 smoke。

## 本版完成

- 新增配置：

```text
JAVA_EXECUTION_CONTRACT_FIXTURE_PATH
MINIKV_CHECKJSON_FIXTURE_PATH
```

- 新增 fixture report 服务：

```text
src/services/upstreamContractFixtures.ts
```

- 新增接口：

```text
GET /api/v1/upstream-contract-fixtures
GET /api/v1/upstream-contract-fixtures?format=markdown
```

- fixture report 会校验：

```text
Java contractVersion / contractDigest / approvalDigest / replayEligibilityDigest
Java replayPreconditionsSatisfied / digestVerificationMode / expectedSideEffects
mini-kv schema_version / command_digest / read_only / execution_allowed
mini-kv side_effects / side_effect_count / wal.durability
```

## 本版不做

- 不真实执行 Java replay POST
- 不真实执行 mini-kv 写命令
- 不把 fixture 当成生产数据
- 不修改 Java 或 mini-kv

## 运行验证

已在 Node 项目内完成：

```text
npm run typecheck
npx vitest run test\config.test.ts test\upstreamContractFixtures.test.ts
npm test
npm run build
Node HTTP smoke with Java v43 fixture + mini-kv v52 fixture
```

smoke 结果保存于：

## 收口判断

v74 把 Node 的 execution-contract smoke 从“手写样本”推进到“读取上游稳定 fixture”。后续如果 Java 或 mini-kv 样本字段变动，Node 能先在 fixture report 中发现，而不是等到操作链路中后段才暴露。
