## 本版目标

Node 接入 mini-kv v47：

```text
KEYSJSON [prefix]
```

并对外提供只读路由：

```text
GET /api/v1/mini-kv/keys?prefix=orderops:
```

Dashboard 增加 key inventory 面板，展示 prefix、key_count、truncated、limit、keys。

## 本版实现

- `src/clients/miniKvClient.ts`
  - 新增 `MiniKvKeysJson`
  - 新增 `keysJson(prefix?)`
  - 新增 `parseMiniKvKeysJson`
  - `validateRawGatewayCommand` 允许 `KEYSJSON`

- `src/routes/miniKvRoutes.ts`
  - 新增 `GET /api/v1/mini-kv/keys`
  - 使用 `UPSTREAM_PROBES_ENABLED` 保护
  - prefix 只允许安全字符

- `src/services/opsSnapshotService.ts`
  - mini-kv probe 新增 `keyInventory`
  - message 增加 `key_inventory=available/unavailable`

- `src/services/upstreamOverview.ts`
  - mini-kv signals 新增 key inventory 可用性、count、truncated、limit、sample keys

- `src/ui/dashboard.ts`
  - mini-kv 卡片新增 Key Inventory 查询入口
  - Dashboard 只展示 key 名，不读取 value

## 本版不做

```text
不新增 SET / DEL 操作能力
不做 key 编辑器
不展示 value 明文
不做大规模 key scan
不修改 Java 项目
不修改 mini-kv 项目
```

### 04 安全 smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
PORT=4198
```

访问：

```text
GET /api/v1/mini-kv/keys?prefix=orderops:
```

结果：

```text
HTTP 403
error=UPSTREAM_PROBES_DISABLED
```

意义：默认安全环境不会误触碰 mini-kv `KEYSJSON`。

### 05 mock mini-kv v47 只读 smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=false
ORDER_PLATFORM_URL=http://127.0.0.1:18085
MINIKV_HOST=127.0.0.1
MINIKV_PORT=16385
PORT=4198
```

本轮只启动本地 TCP mock 模拟 mini-kv v47，不启动真实 mini-kv；同时启动本地 HTTP mock 避免 Java offline 干扰 Dashboard。

结果：

```text
GET /api/v1/mini-kv/keys?prefix=orderops:
command=KEYSJSON orderops:
count=3
truncated=false
keys=orderops:1,orderops:2,orderops:cart
```

overview 结果：

```text
keyInventory=true
count=3
truncated=false
```

意义：Node v58 能只读代理 mini-kv v47 `KEYSJSON`，并把 key inventory 写入 upstream overview。
