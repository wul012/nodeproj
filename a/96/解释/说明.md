# Node v96 说明：Upstream production evidence intake

## 本版目标

v96 按 `docs/plans/v95-production-gap-roadmap.md` 推进，在确认 Java v45 与 mini-kv v54 已完成后，新增 Node 侧只读上游生产证据接入。

本版不启动 Java / mini-kv，不执行 Java replay，不执行 mini-kv 写命令。Node 只读取样本证据，把 Java ops evidence 与 mini-kv STORAGEJSON 统一成控制面可消费的 evidence intake。

## 本版新增能力

- 新增仓库内生产证据样本。
- 新增配置项：
  - `JAVA_OPS_EVIDENCE_FIXTURE_PATH`
  - `MINIKV_STORAGE_EVIDENCE_FIXTURE_PATH`
- 新增 endpoint：`/api/v1/upstreams/production-evidence-intake`。
- 检查 Java `readOnly=true`、`executionAllowed=false`、`realReplayAllowedByEvidence=false`。
- 检查 mini-kv `read_only=true`、`execution_allowed=false`、`order_authoritative=false`。
- 输出 JSON / Markdown，并归档 smoke 与截图。
