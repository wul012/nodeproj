# v36 说明：Promotion Handoff Closure Verification

## 本版目标

v36 在 v35 的 `promotion handoff closure` 后面增加最终复验层：

- 新增 `GET /api/v1/ops/promotion-archive/handoff-closure/verification`
- 新增 `GET /api/v1/ops/promotion-archive/handoff-closure/verification?format=markdown`
- Dashboard 新增 `Closure Verification` 和 `Closure Verification Report`
- 代码讲解新增 `代码讲解记录/40-ops-promotion-handoff-closure-verification-v36.md`

## 核心改动

`src/services/opsPromotionArchiveBundle.ts` 新增：

- `OpsPromotionHandoffClosureVerification`
- `OpsPromotionHandoffClosureVerificationItem`
- `createOpsPromotionHandoffClosureVerification`
- `renderOpsPromotionHandoffClosureVerificationMarkdown`

复验逻辑会重新调用：

```ts
const expectedClosure = createOpsPromotionHandoffClosure({
  receipt: input.receipt,
  receiptVerification: input.receiptVerification,
});
```

然后对比 closure digest、covered fields、receipt/certificate/package/seal digest、decision、verification、nextActions 和 7 个 closure item。

## 对 Java / mini-kv 的影响

本轮没有启动、停止、构建、测试或修改 Java 高并发项目和 mini-kv。

Node smoke 使用了安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此不会主动探测或调用 Java / mini-kv 上游。
