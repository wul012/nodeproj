# Node v73：execution-contract mismatch diagnostics

## 来源

本版依照 `docs/plans/v71-post-execution-contract-roadmap.md` 推进，由 Node v72 的 execution-contract archive bundle 衍生。

v72 已能把 execution gate archive、gate preview、handoff bundle、archive verification、Java contractDigest 和 mini-kv CHECKJSON contract 组织成归档引用链。

v73 的目标是：当这条链路任何一环不一致时，输出清晰的 mismatch diagnostics。

## 本版完成

- 新增 `OperationApprovalExecutionContractDiagnostics`
- 新增接口：

```text
GET /api/v1/operation-approval-execution-gate-archives/:archiveId/execution-contract-diagnostics
GET /api/v1/operation-approval-execution-gate-archives/:archiveId/execution-contract-diagnostics?format=markdown
```

- diagnostics 覆盖：

```text
archiveDigest mismatch
gateDigest vs archived preview mismatch
handoff bundle digest mismatch
archive verification mismatch
Java contractDigest / precondition / verification mode mismatch
mini-kv CHECKJSON command_digest / read_only / execution_allowed mismatch
archive summary vs archived preview snapshot mismatch
```

- 正常 archive 会输出：

```text
valid=true
diagnosticCount=0
diagnostics=[]
```

- 测试里通过手工篡改 archive clone 验证了 Java 与 mini-kv 的 mismatch code。

## 本版不做

- 不自动修复 mismatch
- 不绕过 reviewer decision
- 不把 warning 降级成通过
- 不新增真实 execution endpoint
- 不调用 Java replay POST
- 不执行 mini-kv 写命令

## 收口判断

v73 让 execution-contract 归档链路具备可定位能力：不只是知道 verification 失败，还能看到是哪类 digest、哪个 contract 字段、哪段 archive snapshot 不一致。它仍然是执行前诊断能力，不进入真实执行。
