# OrderOps Node V5 运行调试归档

## 本版目标

V5 增加“本地动作计划 / dry-run 预检”：

- 先查看某个操作会经过哪条 Node 路由。
- 先查看如果真实执行会访问 Java HTTP 还是 mini-kv TCP。
- 先查看当前 `UPSTREAM_ACTIONS_ENABLED` 是否允许真实上游调用。
- 预检接口本身不访问 Java，也不连接 mini-kv。

### 2. dry-run 只读配置，不连上游

文件：`src/services/actionPlanner.ts`

```ts
const allowed = config.upstreamActionsEnabled;
```

```ts
return {
  dryRun: true,
  action: definition.action,
  target: definition.target,
  allowed,
  ...(allowed ? {} : { blockedBy: "UPSTREAM_ACTIONS_ENABLED=false" }),
  wouldCall: definition.upstream(config),
  safety: {
    upstreamProbesEnabled: config.upstreamProbesEnabled,
    upstreamActionsEnabled: config.upstreamActionsEnabled,
  },
};
```

这里没有 `fetch`，没有 `net.createConnection`，没有调用 `OrderPlatformClient` 或 `MiniKvClient`。

## 运行调试结果

- `npm run typecheck`：通过。
- `npm test`：6 个测试文件、19 个测试通过。
- `npm run build`：通过。
- `/health`：返回 `status: ok`。
- `/api/v1/runtime/config`：确认 `upstreamProbesEnabled=false`、`upstreamActionsEnabled=false`。
- `POST /api/v1/action-plans`：
  - `order-create` 返回 `dryRun=true`、`allowed=false`、`blockedBy=UPSTREAM_ACTIONS_ENABLED=false`。
  - `kv-set` 返回 `dryRun=true`、`allowed=false`、`blockedBy=UPSTREAM_ACTIONS_ENABLED=false`。
- `/api/v1/action-plans/catalog`：返回 `order-platform, mini-kv` 两组，共 11 个动作。
- 首页 HTML 包含 `planAction` 和 `/api/v1/action-plans/catalog`。

## 对另外两个项目的影响

本次 V5 调试没有启动、停止、构建、测试或修改：

- `D:\javaproj\advanced-order-platform`
- `D:\C\mini-kv`

Node smoke 只访问了 `orderops-node` 自己的接口，没有访问真实代理路由。

## 本版一句话

V5 把“真实操作前先想清楚会发生什么”做成了 API 和 Dashboard 入口，为后续权限、审批、确认弹窗、审计持久化打基础。
