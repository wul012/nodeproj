# Node v76：Dashboard fixture diagnostics panel

## 来源

本版依照 `docs/plans/v75-post-fixture-diagnostics-roadmap.md` 推进，由 Node v75 的 fixture drift diagnostics 衍生。

v75 已经能通过接口输出 fixture report 与 drift diagnostics。v76 把这两类只读证据放进 Dashboard，方便运行调试时直接观察 Java / mini-kv fixture 状态。

## 本版完成

- Dashboard 新增 `Fixture Report`、`Java Contract Fixture`、`mini-kv Fixture Drift` 三块只读面板。
- 面板展示 fixture report 的 `valid`、`fixtureDigest`、Java fixture status、mini-kv fixture status。
- 面板展示 Java execution-contract 的 `contractDigest`、`replayPreconditionsSatisfied`、Java checks 通过比例、expected side effects 数量。
- 面板展示 mini-kv CHECKJSON 的 `commandDigest`、`read_only`、`execution_allowed`、`side_effect_count`。
- 面板展示 drift diagnostics 的 issue count、error count、missing mapping count，并提供 JSON / Markdown 链接。
- Audit 操作区新增 `Fixture Diagnostics`、`Fixture Report Markdown`、`Fixture Drift Markdown` 三个只读按钮。

## 本版不做

- 不新增真实 execution endpoint
- 不调用 Java replay POST
- 不执行 mini-kv `SET` / `DEL` / `EXPIRE`
- 不修改 Java / mini-kv
- 不把 Dashboard 面板做成审批入口

## 收口判断

v76 没有扩大上游执行能力，只把 v74-v75 已有的 fixture report / drift diagnostics 做成 Dashboard 可读面板。现在本地运行时可以直接看到 fixture 是否有效、Java / mini-kv 关键 digest 是否可用，以及 drift diagnostics 是否发现字段漂移。
