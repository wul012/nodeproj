# Node v64 说明：Approval evidence report + verification

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v64：Approval evidence report + verification
```

本版只使用 Node v62/v63 本地证据，不等待 Java v40 与 mini-kv v49，也不新增真实执行入口。

## 本版目标

新增 approval evidence report：

```text
approval request
approval decision
preflightDigest
previewDigest
decisionDigest
evidenceDigest
verification checks
upstreamTouched=false proof
```

## 本版实现

- `src/services/operationApprovalDecision.ts`
  - 新增 `getByRequest(requestId)`，支持从 request 查找 decision

- `src/services/operationApprovalEvidence.ts`
  - 新增 `createOperationApprovalEvidenceReport`
  - 新增 `createOperationApprovalEvidenceVerification`
  - 新增 `renderOperationApprovalEvidenceMarkdown`
  - 新增 `renderOperationApprovalEvidenceVerificationMarkdown`
  - 复核 request preview digest、decision digest、evidence digest、upstream untouched

- `src/routes/operationApprovalEvidenceRoutes.ts`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/evidence`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/evidence?format=markdown`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/verification`
  - 新增 `GET /api/v1/operation-approval-requests/:requestId/verification?format=markdown`

- `src/app.ts`
  - 注册 approval evidence routes

- `src/ui/dashboard.ts`
  - 新增 `Evidence Report`
  - 新增 `Verify Evidence`

- `test/operationApprovalEvidence.test.ts`
  - 覆盖缺少 decision 的 incomplete evidence
  - 覆盖 approved evidence report
  - 覆盖 verification valid=true
  - 覆盖 Markdown report

### 04 默认安全 evidence smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

流程：

```text
POST /api/v1/operation-intents                  action=kv-set
POST /api/v1/operation-approval-requests
GET  /api/v1/operation-approval-requests/:id/evidence
GET  /api/v1/operation-approval-requests/:id/verification
```

结果：

```text
approvalStatus=rejected
evidenceState=missing-decision
verificationValid=false
decisionPresent=false
```

意义：默认安全环境不会产生可误用的完整 evidence。

### 05 mock mini-kv approved evidence smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=true
MINIKV_HOST=127.0.0.1
MINIKV_PORT=<本地 mock TCP 端口>
```

本轮只启动本地 mock mini-kv TCP 服务，不启动真实 mini-kv，不启动真实 Java。

流程：

```text
POST /api/v1/operation-intents                         action=kv-set
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches                      mode=dry-run
POST /api/v1/operation-approval-requests
POST /api/v1/operation-approval-requests/:id/decision decision=approved
GET  /api/v1/operation-approval-requests/:id/evidence?format=markdown
GET  /api/v1/operation-approval-requests/:id/verification
```

结果：

```text
evidenceState=approved
verificationValid=true
upstreamTouched=false
expectedSideEffects=mini-kv.store-would-mutate,mini-kv.wal-would-be-touched
```

mock mini-kv 收到：

```text
COMMANDSJSON
KEYSJSON orderops:
EXPLAINJSON SET orderops:preview preview-value
```
