# Node v60 说明：Preflight Markdown report + digest

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v60：Preflight Markdown report + digest
```

## 本版目标

把 v59 的 preflight JSON 固化成更适合人工审阅、截图归档和后续复核的报告：

```text
JSON report
Markdown report
SHA-256 preflightDigest
verification endpoint
```

本版仍然只做 Node 控制面能力，不执行真实上游动作，不修改 Java 项目，不修改 mini-kv 项目。

## 本版实现

- `src/services/operationPreflightReport.ts`
  - 新增 `OperationPreflightReport`
  - 新增 `OperationPreflightReportVerification`
  - 使用 `stableJson` + SHA-256 生成 `preflightDigest`
  - Markdown 渲染 report 和 verification
  - report state 分为 `blocked`、`review-required`、`ready`

- `src/routes/operationPreflightRoutes.ts`
  - 新增 `GET /api/v1/operation-intents/:intentId/preflight/report`
  - 新增 `GET /api/v1/operation-intents/:intentId/preflight/report?format=markdown`
  - 新增 `GET /api/v1/operation-intents/:intentId/preflight/verification`
  - verification 也支持 `format=markdown`

- `src/ui/dashboard.ts`
  - Intent 操作区新增 `Report`
  - Intent 操作区新增 `Verify Report`
  - Report 默认请求 Markdown，并显示在 Dashboard 输出框

- `test/operationPreflight.test.ts`
  - 覆盖 report service digest
  - 覆盖 report verification
  - 覆盖 JSON report route
  - 覆盖 Markdown report route
  - 覆盖 verification route

- `README.md`
  - 补充 report / verification endpoint

- `代码讲解记录/64-operation-preflight-report-v60.md`
  - 新增本版代码讲解

### 04 默认安全 report smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
PORT=4610
```

流程：

```text
POST /api/v1/operation-intents       action=kv-set
GET  /api/v1/operation-intents/:id/preflight/report?keyPrefix=orderops:
GET  /api/v1/operation-intents/:id/preflight/verification?keyPrefix=orderops:
```

结果：

```text
intentStatus=blocked
state=blocked
verificationValid=true
hardBlockers=UPSTREAM_ACTIONS_ENABLED=false,INTENT_STATUS_BLOCKED
```

意义：默认安全环境下 report / verification 仍然完全停在 Node 本地，不触碰上游。

### 05 confirmed dry-run report smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=true
PORT=4611
```

流程：

```text
POST /api/v1/operation-intents       action=kv-status
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches    mode=dry-run
GET  /api/v1/operation-intents/:id/preflight/report
GET  /api/v1/operation-intents/:id/preflight/report?format=markdown
GET  /api/v1/operation-intents/:id/preflight/verification
```

结果：

```text
dispatchStatus=dry-run-completed
state=review-required
preflightDigest=sha256 hex
verificationValid=true
```

意义：Node v60 能把 confirmed intent + dry-run dispatch 生成可读 Markdown report，并且能复核 digest。
