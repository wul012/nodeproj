# v38 说明：Promotion Handoff Completion Verification

## 本版目标

v38 在 v37 的 `handoff completion` 后面增加复验层：

- 新增 `GET /api/v1/ops/promotion-archive/handoff-completion/verification`
- 新增 `GET /api/v1/ops/promotion-archive/handoff-completion/verification?format=markdown`
- Dashboard 新增 `Completion Verification` 和 `Completion Verification Report`
- 代码讲解新增 `代码讲解记录/42-ops-promotion-handoff-completion-verification-v38.md`

## 核心改动

`src/services/opsPromotionArchiveBundle.ts` 新增：

- `OpsPromotionHandoffCompletionVerification`
- `OpsPromotionHandoffCompletionVerificationStep`
- `createOpsPromotionHandoffCompletionVerification`
- `renderOpsPromotionHandoffCompletionVerificationMarkdown`

复验逻辑会重新调用：

```ts
const expectedCompletion = createOpsPromotionHandoffCompletion({
  closure: input.closure,
  closureVerification: input.closureVerification,
});
```

然后对比 completion digest、covered fields、completion name、closure digest、verified closure digest、decision、verification、nextActions 和 5 个 completion step。

## 对 Java / mini-kv 的影响

本轮没有启动、停止、构建、测试或修改 Java 高并发项目和 mini-kv。

Node smoke 使用了安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此不会主动探测或调用 Java / mini-kv 上游。
