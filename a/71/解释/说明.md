# Node v71：execution-contract evidence

## 来源

本版依照 `docs/plans/v68-post-execution-gate-roadmap.md` 推进，并先只读确认：

```text
Java v42：replay-execution-contract 已完成
mini-kv v51：CHECKJSON execution contract 已完成
```

随后在 Node 接入两者的只读执行契约证据。

## 本版完成

- `OrderPlatformClient` 新增 `failedEventReplayExecutionContract`
- `MiniKvClient` 新增 `checkJson`
- approval evidence report 新增：
  - Java execution-contract status / contractDigest / replayPreconditionsSatisfied / digestVerificationMode
  - mini-kv CHECKJSON status / read_only / execution_allowed / durability
- verification 新增：
  - `javaExecutionContractEvidenceValid`
  - `miniKvExecutionContractEvidenceValid`
- execution gate preview 新增：
  - `javaReplayPreconditionsSatisfiedOk`
  - `miniKvCheckReadOnlyOk`
  - `miniKvCheckExecutionAllowedOk`
- handoff bundle 与 Markdown 输出同步展示 execution-contract 证据
- 新增后续计划：`docs/plans/v71-post-execution-contract-roadmap.md`

## 本版不做

- 不新增真实 execution endpoint
- 不调用 Java replay POST
- 不执行 mini-kv SET/DEL/EXPIRE
- 不修改 Java 或 mini-kv 项目

## 收口判断

v71 已把 Java v42 与 mini-kv v51 的只读执行契约接入 Node 证据链。现在 Node 具备“执行前契约可读、可归档、可复核”的能力，但仍没有真实执行能力。
