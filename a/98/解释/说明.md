# Node v98 说明：Access-control readiness profile

## 本版目标

v98 按 `docs/plans/v95-production-gap-roadmap.md` 推进，新增 access-control readiness profile。

本版不实现完整登录系统，而是先把当前生产访问控制缺口列成可查询 evidence：无认证、无 RBAC、无操作人身份、审计读取未保护、mutation 路由未保护、CORS 仍是 wildcard。

## 本版新增能力

- 新增 endpoint：`/api/v1/security/access-control-readiness`。
- 输出当前访问控制状态。
- 定义最小角色模型：viewer/operator/approver/auditor/admin。
- 按 route group 输出生产所需最低角色。
- 明确 `UPSTREAM_ACTIONS_ENABLED=false` 只是补偿性控制，不等于生产权限。
