# v40 说明：Promotion Release Evidence Verification

## 本版目标

v40 在 v39 的 `release evidence` 后面增加复验层：

- 新增 `GET /api/v1/ops/promotion-archive/release-evidence/verification`
- 新增 `GET /api/v1/ops/promotion-archive/release-evidence/verification?format=markdown`
- Dashboard 新增 `Release Evidence Verification` 和 `Release Evidence Verification Report`
- 代码讲解新增 `代码讲解记录/44-ops-promotion-release-evidence-verification-v40.md`

## 核心改动

`src/services/opsPromotionArchiveBundle.ts` 新增：

- `OpsPromotionReleaseEvidenceVerification`
- `OpsPromotionReleaseEvidenceVerificationItem`
- `createOpsPromotionReleaseEvidenceVerification`
- `renderOpsPromotionReleaseEvidenceVerificationMarkdown`

复验逻辑会重新调用：

```ts
const expectedEvidence = createOpsPromotionReleaseEvidence({
  completion: input.completion,
  completionVerification: input.completionVerification,
});
```

然后对比 evidence digest、covered fields、completion/closure digest、decision、verification、nextActions 和 5 个 evidence item。

## 对 Java / mini-kv 的影响

本轮没有启动、停止、构建、测试或修改 Java 高并发项目和 mini-kv。

Node smoke 使用了安全环境变量：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

因此不会主动探测或调用 Java / mini-kv 上游。
