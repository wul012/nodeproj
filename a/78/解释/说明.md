# Node v78：Fixture scenario matrix

## 来源

本版依照 `docs/plans/v75-post-fixture-diagnostics-roadmap.md` 推进，由 Node v77 的 fixture evidence archive snapshot 衍生。

推进前已按规则只读核对 Java v44 和 mini-kv v53：

```text
Java v44 tag：v44失败事件重放execution-contract-blocked-sample
mini-kv v53 tag：第五十三版CHECKJSON读命令稳定样本
Java blocked sample：failed-event-replay-execution-contract-blocked.sample.json
mini-kv read sample：get-orderops-read-contract.json
mini-kv read side_effects=["store_read"]，side_effect_count=1
```

## 本版完成

- 新增 scenario matrix 服务：

```text
src/services/upstreamContractFixtureScenarioMatrix.ts
```

- 新增接口：

```text
GET /api/v1/upstream-contract-fixtures/scenario-matrix
GET /api/v1/upstream-contract-fixtures/scenario-matrix?format=markdown
```

- 新增四个 fixture 路径配置：

```text
JAVA_EXECUTION_CONTRACT_FIXTURE_PATH
JAVA_EXECUTION_CONTRACT_BLOCKED_FIXTURE_PATH
MINIKV_CHECKJSON_FIXTURE_PATH
MINIKV_CHECKJSON_READ_FIXTURE_PATH
```

- matrix 覆盖四个场景：

```text
java-approved-replay-contract
java-blocked-replay-contract
mini-kv-write-checkjson
mini-kv-read-checkjson
```

- blocked / negative sample 被当作诊断样本，只要字段语义符合预期，就计为 valid，不当作失败测试。

## 本版不做

- 不真实执行 Java replay POST
- 不真实执行 mini-kv 写命令
- 不自动修复 fixture
- 不引入 Java / mini-kv 运行时强依赖
- 不开启 `UPSTREAM_ACTIONS_ENABLED` 默认值

## 收口判断

v78 把 Java approved / blocked execution-contract 和 mini-kv write / read CHECKJSON 四类样本汇成同一张矩阵。现在 Node 可以用一个只读接口判断每个场景是否具备 diagnostics 所需字段，并把 drift issue 聚合到 matrix 级别。
