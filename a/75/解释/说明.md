# Node v75：fixture drift diagnostics

## 来源

本版依照 `docs/plans/v71-post-execution-contract-roadmap.md` 推进，由 Node v74 的 fixture-driven smoke 衍生。

v74 已能读取 Java v43 与 mini-kv v52 的稳定样本，并生成 fixture report。v75 在此基础上增加 drift diagnostics，专门定位 fixture 字段漂移。

## 本版完成

- 新增 drift diagnostics 服务：

```text
src/services/upstreamContractFixtureDrift.ts
```

- 新增接口：

```text
GET /api/v1/upstream-contract-fixtures/drift-diagnostics
GET /api/v1/upstream-contract-fixtures/drift-diagnostics?format=markdown
```

- diagnostics 覆盖：

```text
Java contractVersion / contractDigest / approvalDigest / replayEligibilityDigest
Java replayPreconditionsSatisfied / digestVerificationMode / expectedSideEffects
mini-kv schema_version / command_digest / read_only / execution_allowed
mini-kv side_effects / side_effect_count / wal.durability
fixture field -> Node summary field -> diagnostics field 映射
```

- 正常样本会输出：

```text
valid=true
issueCount=0
missingMappingCount=0
issues=[]
```

## 本版不做

- 不要求 Java / mini-kv 同步发版
- 不阻断已有只读观察台
- 不自动修复 fixture
- 不真实执行 Java replay POST
- 不真实执行 mini-kv 写命令

## 收口判断

v75 让 Node 能在 fixture-driven smoke 前先判断样本是否漂移。现在如果上游 fixture 缺字段、类型变化、digest 格式变化、side_effect_count 不匹配，或无法映射到 Node diagnostics 所需字段，Node 会给出明确 issue code。
