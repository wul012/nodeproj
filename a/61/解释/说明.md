# Node v61 说明：Operation execution preview

## 来源计划

本版按最新计划执行：

```text
docs/plans/v59-post-preflight-control-roadmap.md
```

对应版本：

```text
Node v61：Operation execution preview
```

本版依赖已只读确认：

```text
Java v39：失败事件重放 simulation
mini-kv v48：EXPLAINJSON 命令解释核心
```

## 本版目标

在 v59 preflight 和 v60 report/digest 基础上，新增 execution preview：

```text
preflight digest
Java replay simulation evidence
mini-kv EXPLAINJSON evidence
would-call upstream summary
expected side effects
final blockers / warnings
```

本版仍然不执行 Java replay，不执行 mini-kv 写命令，只做只读预演。

## 本版实现

- `src/clients/orderPlatformClient.ts`
  - 新增 `OrderPlatformFailedEventReplaySimulation`
  - 新增 `failedEventReplaySimulation(failedEventId)`

- `src/clients/miniKvClient.ts`
  - 新增 `MiniKvExplainJson`
  - 新增 `explainJson(command)`
  - 新增 `parseMiniKvExplainJson`
  - `validateRawGatewayCommand` 允许 `EXPLAINJSON`

- `src/routes/orderPlatformRoutes.ts`
  - 新增 `GET /api/v1/order-platform/failed-events/:failedEventId/replay-simulation`
  - 使用 `UPSTREAM_PROBES_ENABLED` 保护

- `src/routes/miniKvRoutes.ts`
  - 新增 `GET /api/v1/mini-kv/explain?command=...`
  - 使用 `UPSTREAM_PROBES_ENABLED` 保护

- `src/services/actionPlanner.ts`
  - 新增 `failed-event-replay-simulation`
  - 归类为 `read` + `probes`

- `src/services/operationExecutionPreview.ts`
  - 新增 `OperationExecutionPreviewService`
  - 汇总 preflight report digest、Java simulation、mini-kv explain、expected side effects、hard blockers、warnings

- `src/routes/operationPreflightRoutes.ts`
  - 新增 `GET /api/v1/operation-intents/:intentId/execution-preview`

- `src/ui/dashboard.ts`
  - action 选项新增 `Order: replay simulation`
  - Intent 操作区新增 `Execution Preview`

## 本版不做

```text
不真实执行 Java replay
不真实执行 mini-kv raw command
不批量处理多个 intent
不绕过 Java / mini-kv 自己的安全规则
不修改 Java 项目
不修改 mini-kv 项目
```

### 04 默认安全 execution-preview smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
PORT=4620
```

流程：

```text
POST /api/v1/operation-intents       action=kv-set
GET  /api/v1/operation-intents/:id/execution-preview?key=orderops:preview&value=preview-value
```

结果：

```text
state=blocked
hardBlockers=UPSTREAM_ACTIONS_ENABLED=false,INTENT_STATUS_BLOCKED
miniKvExplain=skipped
```

意义：默认安全环境不会触碰 Java 或 mini-kv。

### 05 mock Java + mock mini-kv execution-preview smoke

环境：

```text
UPSTREAM_PROBES_ENABLED=true
UPSTREAM_ACTIONS_ENABLED=true
ORDER_PLATFORM_URL=http://127.0.0.1:18091
MINIKV_HOST=127.0.0.1
MINIKV_PORT=16411
PORT=4621
```

本轮只启动本地 mock Java HTTP 服务和 mock mini-kv TCP 服务，不启动真实 Java / mini-kv。

Java 流程：

```text
POST /api/v1/operation-intents       action=failed-event-replay-simulation
POST /api/v1/operation-intents/:id/confirm
GET  /api/v1/operation-intents/:id/execution-preview?failedEventId=42
```

Java 结果：

```text
state=review-required
javaReplaySimulation=available
expectedSideEffects=order-event-replayed,outbox-message-created,java.failed-event.would-replay,java.outbox.would-publish,java.failed-event.status-would-change
```

mini-kv 流程：

```text
POST /api/v1/operation-intents       action=kv-set
POST /api/v1/operation-intents/:id/confirm
POST /api/v1/operation-dispatches    mode=dry-run
GET  /api/v1/operation-intents/:id/execution-preview?key=orderops:preview&value=preview-value&keyPrefix=orderops:
```

mini-kv 结果：

```text
state=review-required
miniKvCommandExplain=available
expectedSideEffects=mini-kv.store-would-mutate,mini-kv.wal-would-be-touched
```
