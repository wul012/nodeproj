# Node v67：digest-aware upstream evidence verification

## 来源

本版依照 `docs/plans/v65-post-upstream-evidence-roadmap.md` 推进，目标是把 Java v41 与 mini-kv v50 新增的只读证据字段接入 Node 的 approval evidence verification。

## 本版完成

- Java approval-status 证据接入 `evidenceVersion`、`approvalDigest`、`replayEligibilityDigest`
- mini-kv EXPLAINJSON 证据接入 `schema_version`、`command_digest`、`side_effect_count`
- `operationApprovalEvidence` 的 summary、verification checks、Markdown 输出同步展示这些字段
- `operationApprovalHandoffBundle` 的 summary 与 Markdown 输出同步展示这些字段
- verification 新增：
  - `javaApprovalDigestEvidenceValid`
  - `miniKvCommandDigestEvidenceValid`
  - `miniKvSideEffectCountMatches`

## 本版不做

- 不调用 Java replay POST
- 不执行 mini-kv SET/DEL/EXPIRE
- 不修改 Java 或 mini-kv 项目
- 不把 digest mismatch 自动修复为通过
