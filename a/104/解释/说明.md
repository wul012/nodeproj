# Node v104 归档说明

## 本版目标

本版完成 access guard audit context，把 v101 的 dry-run access guard 结果写入请求审计事件。

## 新增能力

- `AuditEvent` 新增 `accessGuard` 字段。
- `src/app.ts` 在 `onRequest` 计算 access guard 后，将其缓存到请求上下文。
- `onResponse` 写审计事件时同时写入：
  - `routeGroup`
  - `requiredRole`
  - `matchedRoles`
  - `wouldDeny`
  - `reason`
- `FileBackedAuditStore` 读取旧事件仍兼容，读取新事件能保留 `accessGuard`。

## 安全边界

- 仍然是 `mode=dry-run`
- 仍然是 `rejectsRequests=false`
- 不启用真实登录
- 不读取 secret
- 不执行 Java replay
- 不执行 mini-kv 写命令
