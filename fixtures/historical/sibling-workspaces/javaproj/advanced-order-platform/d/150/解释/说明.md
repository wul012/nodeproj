# Java v150 abort/rollback semantics contract echo

## 本版做了什么

Java v150 接入 Node v326 的 abort/rollback semantics contract intake，新增只读 echo receipt：

```text
managedAuditSandboxEndpointCredentialResolverAbortRollbackSemanticsContractEchoReceipt
```

它证明 Java 已识别最终 prerequisite 的契约字段、禁止字段、拒绝原因和 no-go 边界，同时继续保持不执行 SQL、部署、回滚、HTTP/TCP、runtime shell、provider/client、ledger/schema 写入、mini-kv 写入或自动启动上游进程。

## 主要改动

```text
ReleaseApprovalSandboxEndpointCredentialResolverAbortRollbackSemanticsContractCatalog.java
ReleaseApprovalSandboxEndpointCredentialResolverAbortRollbackSemanticsContractEchoRecords.java
ReleaseApprovalSandboxEndpointCredentialResolverAbortRollbackSemanticsContractEchoSupport.java
ReleaseApprovalSandboxEndpointCredentialResolverAbortRollbackSemanticsContractSections.java
ReleaseApprovalManagedAuditSandboxEndpointCredentialResolverAbortRollbackSemanticsContractEchoReceiptBuilder.java
```

新 receipt 接入 `ReleaseApprovalRehearsalResponse`、receipt chain、verification hint、warning digest 和 schema field catalog。响应 schema 从 v49 推进到 v50。
