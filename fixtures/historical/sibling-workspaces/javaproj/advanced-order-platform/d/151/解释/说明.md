# Java v151 release approval evidence export hint

## 本版做了什么

Java v151 按 Node v328 plan 判断：Node v329 还没有生成，所以这次不继续扩张 echo support，而是先做一版小范围质量优化。

新增响应块：

```text
evidenceExportHint
```

它告诉后续 Node：优先从当前稳定只读 JSON 响应导出证据，推荐 artifact 为：

```text
release-approval-rehearsal-current.json
```

## 主要改动

```text
ReleaseApprovalRehearsalResponseRecords.java
ReleaseApprovalRehearsalResponse.java
ReleaseApprovalRehearsalEvidenceExportHintBuilder.java
ReleaseApprovalRehearsalResponseBuilder.java
ReleaseApprovalVerificationHintCatalog.java
OpsEvidenceService.java
```

响应 schema 从 v50 推进到 v51，并把 `evidenceExportHint` 加入 schema field catalog。新增 hint 明确当前 endpoint、源 evidence endpoint、导出模式、必查字段和禁止动作。

## 边界

本版仍保持只读：

```text
不读取 credential value
不解析 raw endpoint url
不执行 HTTP/TCP
不写 approval ledger
不执行 schema migration SQL
不触发 deployment / rollback
不自动启动 upstream process
```

## 文件规模

这次没有新增 600+ 行 echo support，属于合理的小质量优化。
