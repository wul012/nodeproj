# Java v131 说明

本版补的是 direct execution-denied echo receipt。

它回应的是 Node v292 的缺口：Java 自己给出一份只读证据，说明这条链路没有执行 fake harness，没有读取 credential，没有解析 raw endpoint，没有写 ledger/SQL/schema，也没有自动启动任何运行时。

保留的源证据仍是 implementation-plan echo receipt（schema v36）。新增的 rehearsal response schema 升到 v37，只是为了容纳这条新 receipt。
