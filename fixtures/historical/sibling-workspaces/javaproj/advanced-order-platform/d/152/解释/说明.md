# Java v152 input-hardening decision echo

## 本版做了什么

Java v152 消费 Node v329 的 implementation candidate gate / input-hardening decision，并在 release approval rehearsal 响应里新增：

```text
inputHardeningDecisionEcho
```

这版确认：Java v151 已经提供稳定只读 evidence export，Java 侧满足 Node v329 提出的 `java-stable-evidence-export` 要求，但 Node v330 仍要等待并消费两边上游证据后才能继续判断。

## 主要改动

```text
ReleaseApprovalRehearsalInputHardeningDecisionEchoBuilder.java
ReleaseApprovalRehearsalResponseRecords.java
ReleaseApprovalRehearsalResponse.java
ReleaseApprovalRehearsalResponseBuilder.java
ReleaseApprovalVerificationHintCatalog.java
OpsEvidenceService.java
```

响应 schema 从 v51 推进到 v52，并把 `inputHardeningDecisionEcho` 加入 schema field catalog。

## 边界

本版仍保持只读：

```text
不读取 credential value
不解析 raw endpoint URL
不实例化 provider/client
不发 HTTP/TCP
不打开 managed audit connection
不写 approval ledger / schema
不执行 SQL / deployment / rollback
不自动启动 upstream process
```
