# v87 运行说明

## 1. 版本目标

Java v87 按 Node `v227-post-evidence-checklist-roadmap.md` 推进，只新增 release approval rehearsal 的 `managedAuditSandboxConnectionOperatorHandoffMarker`。

它承接 Node v228 `managed-audit-manual-sandbox-connection-operator-packet.v1`，给 Node v229 packet verification 做 Java 侧只读 handoff marker：

- 识别 owner approval artifact id 字段
- 识别 sandbox credential handle name 字段
- 识别 schema rehearsal id 字段
- 识别 rollback path id、timeout budget、manual abort marker
- 不读取 credential value
- 不打开 managed audit sandbox connection
- 不执行 schema migration SQL
- 不写 approval ledger / managed audit store

## 2. 拆分方式

新增 `ReleaseApprovalManagedAuditSandboxConnectionOperatorHandoffMarkerBuilder`，没有把 marker 逻辑继续堆进 `OpsEvidenceService`。

字段按 6 组拆分：

- sandbox connection window
- operator packet
- credential boundary
- schema rehearsal
- rollback path
- Java execution

这样避免新增长布尔构造链，也让 Node v229 后续可以按边界校验。

## 3. 响应变化

新增字段：

```text
managedAuditSandboxConnectionOperatorHandoffMarker
```

schema 变化：

```text
verificationHint.responseSchemaVersion=java-release-approval-rehearsal-response-schema.v17
```

默认无完整 upstream header 时，上游 sandbox guard 未 ready：

```text
managedAuditSandboxConnectionOperatorHandoffMarker.readyForNodeV229ManualSandboxConnectionPacketVerification=false
managedAuditSandboxConnectionOperatorHandoffMarker.markerWarnings=[
  NODE_V229_SOURCE_SANDBOX_ADAPTER_APPROVAL_SCHEMA_GUARD_RECEIPT_NOT_READY
]
```

传入完整 release approval rehearsal header 后：

```text
managedAuditSandboxConnectionOperatorHandoffMarker.readyForNodeV229ManualSandboxConnectionPacketVerification=true
managedAuditSandboxConnectionOperatorHandoffMarker.markerWarnings=[]
```

但这只表示 Node v229 可以做 packet verification，不代表可以连接 sandbox 或进入生产审计。

## 4. 边界不变

- Java 不读取 credential value
- Java 不打开 external / sandbox managed audit connection
- Java 不执行 schema migration SQL
- Java 不写 approval ledger
- Java 不写 managed audit store
- Java 不触发 deployment / rollback / restore
- Java 不启动 Node、mini-kv 或外部 audit 服务

## 5. 验证

应跑：

```text
mvn -q -DskipTests compile
mvn -q '-Dtest=OpsEvidenceServiceTests,OpsOverviewIntegrationTests' test
mvn -q -DskipTests package
git diff --check
```

## 6. 清理

构建产物 `target/` 会在最终收口时删除；本次不需要启动后台进程。
