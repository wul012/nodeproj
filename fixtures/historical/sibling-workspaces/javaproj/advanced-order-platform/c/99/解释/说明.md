# Java v99 说明：manual sandbox connection precheck packet echo receipt

## 计划依据

本轮依据最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v245-post-sandbox-precheck-roadmap.md
```

计划要求 Java v99 做：

```text
manual sandbox connection precheck packet echo receipt
只读回显 Node v245 的 owner approval artifact、credential handle review、schema migration rehearsal id、operator window、rollback path、abort marker、timeout policy
不写 ledger
不执行 SQL
不打开 managed audit connection
```

## 合理性判断

合理。Node v245 已经生成真实 sandbox connection 前的 precheck packet，但这个 packet 仍不是连接许可。Java v99 只负责把 Java 侧看到的字段、数量、digest 和 no-write / no-start 边界回显给 Node v246 核对。

本版继续用独立 builder/helper 承载新增 receipt，不把字段继续塞进 `OpsEvidenceService` 主流程；builder 内部用 flags record 表达 shape、field echo 和 execution boundary，避免新增位置布尔参数串。
