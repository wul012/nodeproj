# Java v102 说明：disabled adapter client precheck echo receipt

## 计划依据

本轮先读取并对齐最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v252-post-disabled-adapter-client-precheck-roadmap.md
```

计划要求 Java v102 做：

```text
disabled adapter client precheck echo receipt
只读回显 Node v252 的 env handle、opt-in gate、failure taxonomy、dry-run response shape 和 no-go 条件
不写 approval ledger
不执行 SQL
不打开 managed audit connection
不读取 credential value
```

## 合理性判断

合理。Node v252 已把真实 adapter client 之前的 disabled precheck 形状固定下来，Java v102 适合只做回显 receipt，让 Node v254 后续可以校验 Java、Node、mini-kv 三侧边界是否一致。

本轮没有把字段继续堆回 `OpsEvidenceService`，而是沿用 v94-v101 的拆分方式，把新 receipt 构建逻辑放入独立 builder。

## 本版目标

新增一个只读 receipt：

```text
managedAuditSandboxConnectionDisabledAdapterClientPrecheckEchoReceipt
```

它证明 Java 已回显：

```text
Node v252 profileVersion
disabled-adapter-client-precheck-ready
5 个 required env handles
6 个 failure class codes
10 个 dry-run response fields
8 个 reused no-go conditions
ORDEROPS_MANAGED_AUDIT_ADAPTER_CLIENT_ENABLED opt-in gate
```

同时证明 Java 没有做：

```text
instantiate managed audit adapter client
read credential value
send external managed audit request
open managed audit sandbox connection
execute schema migration SQL
write approval ledger
start Java / mini-kv / external audit service
request mini-kv write permission
```
