# v89 运行说明

## 1. 版本目标

Java v89 按 Node `v231-post-preflight-verification-roadmap.md` 做低风险质量优化：新增 `ContextHeaderField` record，收敛 release approval rehearsal 中 header `value/source/echoed` 成对字段的重复构造。

本版不新增响应字段，不修改 schema version，不改变 JSON contract，不新增 approval 行为，不写 ledger，不执行 SQL。

```

```

## 3. ContextHeaderField 做什么

`ContextHeaderField` 把三件事收成一个小对象：

```text
value   -> header 值或 placeholder
source  -> header 名或 NOT_SUPPLIED
echoed  -> 是否真的从 header echo 到
```

并提供：

```text
addMissingWarning(...)
allEchoed(...)
```

这样 request / operator / CI / artifact / live / audit persistence / approval record 这些 hint 不再反复手写：

```text
valueOrPlaceholder(...)
sourceFor(...)
normalizedX != null
```

## 4. 边界不变

- Java 不新增 approval decision
- Java 不写 approval ledger
- Java 不持久化 approval record
- Java 不写 managed audit store
- Java 不执行 SQL
- Java 不触发 deployment / rollback / restore
- Java 不读取 credential value
- Java 不打开 managed audit connection
- Java 不自动启动 Node、mini-kv 或外部 audit 服务
