# v82 运行说明

## 1. 版本目标

Java v82 按 Node 最新计划 `docs/plans/v223-post-external-adapter-readiness-roadmap.md` 推进，在 `/api/v1/ops/release-approval-rehearsal` 新增 `managedAuditSandboxAdapterApprovalSchemaGuardReceipt`。

它承接 Node v224 `managed-audit-sandbox-adapter-dry-run-plan.v1`，给 Node v225 sandbox adapter dry-run package 提供 Java 侧只读 approval/schema guard：

```text
Node v224 sandbox adapter dry-run plan
 -> Java v82 sandbox adapter approval/schema guard receipt
 -> Node v225 sandbox adapter dry-run package
```

本版本不执行真实外部 managed audit 连接，不读取 credential value，不执行 schema migration SQL，不写 Java approval ledger 或 managed audit store，也不打开生产审计窗口。

## 2. 新增字段

`ReleaseApprovalRehearsalResponse` 顶层新增：

```java
RehearsalManagedAuditSandboxAdapterApprovalSchemaGuardReceipt
        managedAuditSandboxAdapterApprovalSchemaGuardReceipt
```

核心字段按边界分组：

```text
nodeV224SandboxPlan
 -> 记录 Node v224 plan 是 read-only，未连接 managed audit，未读取或保存 credential，未执行 schema migration

ownerApprovalBoundary
 -> 要求 owner approval artifact，但 Java 不提供 artifact、不创建 approval decision、不写 ledger

schemaRehearsalBoundary
 -> 要求 schema migration rehearsal 和 checklist，但 Java 不允许执行 SQL 或 apply migration

credentialBoundary
 -> 只要求 sandbox credential handle，固定 handle 名为 ORDEROPS_MANAGED_AUDIT_SANDBOX_CREDENTIAL_HANDLE，不要求也不读取 credential value

executionBoundary
 -> external connection、service start、managed audit store write、SQL、deployment、rollback、restore、production audit window 全部为 false

qualityGateBoundary
 -> 证明 v82 使用 builder/helper 拆分，避免长 boolean constructor chain，OpsEvidenceService 只做接线
```

## 3. Ready 条件

默认缺少完整 Node v210 approval binding header 时，上游 v81 receipt 不 ready，因此 v82 返回：

```text
readyForNodeV225SandboxAdapterDryRunPackage=false
guardWarnings=[NODE_V225_SOURCE_EXTERNAL_ADAPTER_MIGRATION_GUARD_RECEIPT_NOT_READY]
```

传入完整 header 后，v81 source receipt ready，且 v82 所有 no-write/no-connection/no-credential/no-SQL 条件保持成立时：

```text
readyForNodeV225SandboxAdapterDryRunPackage=true
readyForProductionAudit=false
readyForProductionWindow=false
nodeMayTreatAsProductionAuditRecord=false
```

这只允许 Node v225 继续生成 sandbox adapter dry-run package，不代表生产审计、真实外部连接或生产窗口授权。

## 5. 质量拆分

v82 新增：

```java
ReleaseApprovalManagedAuditSandboxAdapterApprovalSchemaGuardReceiptBuilder
```

它负责：

- 构造 v82 receipt
- 维护 v82 warning digest input 名称
- 维护 v82 proof claims
- 维护 v82 Node verification actions
- 维护 v82 no-write/no-credential/no-connection/no-schema 判断
- 生成 v82 guard digest

`OpsEvidenceService` 只复用这个 builder 做接线和组合，避免继续把 v82 细节塞进主服务。
