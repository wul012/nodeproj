# Java v103 说明：fake transport dry-run packet echo marker

## 计划依据

本轮先读取并对齐最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v255-post-fake-transport-dry-run-roadmap.md
```

计划要求 Java v103 做：

```text
fake transport dry-run packet echo marker
只读回显 Node v255 fake transport packet 的 request / response / timeout / cleanup 边界
不写 ledger
不执行 SQL
不读取 credential value
不打开 managed audit connection
不创建临时文件或目录
不启动 Java / mini-kv / external audit service
```

## 合理性判断

合理。Node v255 已把 fake transport dry-run packet 固化成只读验证包，Java v103 适合做 echo marker，证明 Java 能识别这份包的形状和安全边界，但不把它升级成真实 adapter connection。

本轮继续沿用 v94-v102 的拆分方式：`OpsEvidenceService` 只增加版本和 Node profile 常量，具体 marker 构建放入独立 builder，避免主类继续膨胀。

## 清理记录

验证产生的 `target/` 会在最终收口前删除；本版不保留临时构建产物，不启动后台服务。

## 一句话总结

v103 给 release approval rehearsal 增加 fake transport dry-run packet echo marker，让 Java 只读回显 Node v255/v256 的 packet 和 archive 边界，供 Node v257 继续校验，同时继续阻断 credential value、真实连接、外部请求、SQL、ledger、临时文件和自动启动。
