# Java v107 运行说明：test-only resolver shell echo marker

## 计划依据

本版依据当前有效计划：

```text
D:\nodeproj\orderops-node\docs\plans\v263-post-disabled-resolver-echo-roadmap.md
```

计划要求 Java v107 只读回显 Node v264 的 credential resolver test-only shell contract。这里的 shell 是 fake in-memory contract，不是真实 secret provider、resolver client 或 external endpoint adapter。

## 合理性判断

合理。Node v264 已经消费 Node v263 的 disabled precheck upstream echo verification，并把下一步限制在 fake-only request / response / failure mapping / guard / fake probe。Java v107 只把这些字段固化为 release approval rehearsal marker，给 Node v265 做 upstream echo verification。

不合理的部分没有做：不实现真实 resolver，不读取 credential value，不解析 raw endpoint URL，不发 external request，不打开 managed-audit connection，不执行 SQL/schema migration，不写 ledger，不自动启动上游。
