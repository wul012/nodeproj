# v88 运行说明

## 1. 版本目标

Java v88 按 Node `v229-post-packet-verification-roadmap.md` 推进，只新增 release approval rehearsal 的 `managedAuditSandboxConnectionPreflightEchoMarker`。

它承接 Node v230 `managed-audit-manual-sandbox-connection-preflight-gate.v1`，给 Node v231 manual sandbox connection preflight verification 做 Java 侧只读 echo marker：

- echo owner approval artifact id 字段
- echo sandbox credential handle name 字段
- echo schema rehearsal id 字段
- echo rollback path id、timeout budget、manual abort marker
- echo manual sandbox window flag name
- 不读取 credential value
- 不打开 managed audit sandbox connection
- 不执行 schema migration SQL
- 不写 approval ledger / managed audit store
- 不自动启动 Java、mini-kv 或外部 audit 服务

## 2. 拆分方式

新增 `ReleaseApprovalManagedAuditSandboxConnectionPreflightEchoMarkerBuilder`，没有把 v88 marker 逻辑堆进 `OpsEvidenceService`。

字段按 6 组拆分：

- manual sandbox connection window
- preflight fields
- credential boundary
- schema rehearsal
- rollback path
- Java execution

builder 内部使用语义 flags record 转成响应 boundary，避免继续新增裸布尔位置参数链。

## 3. 响应变化

新增字段：

```text
managedAuditSandboxConnectionPreflightEchoMarker
```

schema 变化：

```text
verificationHint.responseSchemaVersion=java-release-approval-rehearsal-response-schema.v18
```

默认无完整 upstream header 时，上游 v87 operator handoff marker 未 ready：

```text
managedAuditSandboxConnectionPreflightEchoMarker.readyForNodeV231ManualSandboxConnectionPreflightVerification=false
managedAuditSandboxConnectionPreflightEchoMarker.markerWarnings=[
  NODE_V231_SOURCE_SANDBOX_CONNECTION_OPERATOR_HANDOFF_MARKER_NOT_READY
]
```

传入完整 release approval rehearsal header 后：

```text
managedAuditSandboxConnectionPreflightEchoMarker.readyForNodeV231ManualSandboxConnectionPreflightVerification=true
managedAuditSandboxConnectionPreflightEchoMarker.markerWarnings=[]
```

这只表示 Node v231 可以做 preflight verification，不代表可以连接 sandbox 或进入生产审计。

## 4. Node v230 预检字段

v88 echo marker 固定 Node v230 的 7 个预检字段：

```text
ORDEROPS_MANAGED_AUDIT_OWNER_APPROVAL_ARTIFACT_ID
ORDEROPS_MANAGED_AUDIT_SANDBOX_CREDENTIAL_HANDLE
ORDEROPS_MANAGED_AUDIT_SCHEMA_REHEARSAL_ID
ORDEROPS_MANAGED_AUDIT_ROLLBACK_PATH_ID
timeoutBudgetMs=15000
ORDEROPS_MANAGED_AUDIT_MANUAL_ABORT
ORDEROPS_MANAGED_AUDIT_MANUAL_SANDBOX_WINDOW_APPROVED
```

## 5. 边界不变

- Java 不读取 credential value
- Java 不打开 external / sandbox managed audit connection
- Java 不执行 schema migration SQL
- Java 不写 approval ledger
- Java 不写 managed audit store
- Java 不触发 deployment / rollback / restore
- Java 不启动 Node、mini-kv 或外部 audit 服务
- manual window 默认不开启，且不由 Java 打开
