# Java v106 说明：sandbox endpoint credential resolver disabled precheck echo marker

## 计划依据

最新计划来自：
```text
D:\nodeproj\orderops-node\docs\plans\v260-post-credential-resolver-decision-roadmap.md
```

本版承接 Node v262 的 disabled precheck，只做 Java 只读回显，不实现 credential resolver，不接 secret provider，不读取 credential value，也不解析真实 endpoint。

## 合理性判断

合理。Node v262 已把下一步收敛成 disabled resolver precheck：先确认环境句柄、显式 opt-in gate、失败分类、dry-run 响应形状和 no-go 条件，再让 Node v263 做上游 echo verification。Java v106 只负责把这些边界固化进 release approval rehearsal 响应，让后续链路可以验证“仍未执行 resolver”，而不是提前打开真实连接或凭据访问。

## 本版目标

新增字段：
```text
managedAuditSandboxEndpointCredentialResolverDisabledPrecheckEchoMarker
```

核心回显：
```text
Node v262 disabled precheck profile / endpoint / state
Node v261 upstream echo verification source
6 个 required env handles
2 个 opt-in gates
7 个 failure classes
12 个 dry-run response fields
9 个 inherited no-go conditions
read-only side-effect boundary
```

schema 变化：
```text
source credential resolver decision marker schema: java-release-approval-rehearsal-response-schema.v27
response schema: java-release-approval-rehearsal-response-schema.v28
```

## 边界

本版继续保持不做：
```text
不实现 credential resolver
不实例化 resolver client
不实例化 secret provider
不读取、加载、存储或返回 credential value
不解析 raw endpoint URL
不发送外部请求
不连接 managed audit
不执行 schema migration
不写 approval ledger
不自动启动 Java / mini-kv / external audit service
不把 mini-kv 变成 managed audit storage
```

## 一句话总结

Java v106 把 Node v262 的 disabled credential resolver precheck 固化成只读 echo marker，为 Node v263 校验准备输入，同时继续把 credential、endpoint、连接、SQL、ledger 和自动启动全部挡住。
