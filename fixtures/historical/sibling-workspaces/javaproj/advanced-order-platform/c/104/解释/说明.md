# Java v104 说明：sandbox endpoint handle preflight echo marker

## 计划依据

本轮先读取并对齐最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v257-post-fake-transport-upstream-echo-roadmap.md
```

计划要求 Java v104 做：

```text
sandbox endpoint handle preflight echo marker
只读回显 Node v258 的 endpoint/credential handle review
只读回显 network allowlist / TLS policy / redaction policy / operator window 边界
不读 credential value
不解析 raw endpoint URL
不发送真实 managed audit HTTP 请求
不执行 schema migration
不写 approval ledger
不启动 Java / mini-kv / external audit service
```

## 合理性判断

合理。Node v258 已经完成 sandbox endpoint handle preflight review，但它仍然停在 handle-only review 阶段，没有打开真实 endpoint、credential resolver 或 schema migration。Java v104 适合做一份只读 echo marker，证明 Java 能识别 Node v258 的预检结果和禁止项，同时继续把真实 managed-audit connection、SQL、ledger、auto-start 都挡住。

本轮继续沿用 v94-v103 的拆分方式：`OpsEvidenceService` 只增加版本/profile 常量，不把 marker 构建逻辑塞回主类；具体构建、digest、proof claim 和 no-ledger proof 都放入独立 builder。

v104 给 release approval rehearsal 增加 sandbox endpoint handle preflight echo marker，让 Java 只读回显 Node v258 的 endpoint/credential handle review、network/TLS/redaction/operator-window 边界，供 Node v259 继续校验，同时继续阻断 credential value、raw endpoint、真实连接、外部请求、SQL、ledger 和自动启动。
