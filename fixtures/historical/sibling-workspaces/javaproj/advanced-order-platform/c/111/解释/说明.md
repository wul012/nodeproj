# Java v111 运行说明：credential resolver production-readiness blocked-decision echo receipt

## 计划依据

本版依据当前有效计划：

```text
D:\nodeproj\orderops-node\docs\plans\v268-post-production-readiness-decision-roadmap.md
```

计划要求 Java v111 只读承接 Node v268 的 credential resolver production-readiness decision gate，生成 Java 侧 blocked-decision echo receipt。它不是 resolver 实现，不读取 credential value，不解析 raw endpoint，不发外部请求，不打开 managed audit connection，不写 approval ledger，不执行 SQL，也不做 schema migration 或 auto-start。

## 合理性判断

合理。Node v268 的结论是 blocked：fake-shell archive echo chain 已经对齐，但真实 resolver pre-implementation plan 仍缺 10 个必要边界。Java 现在应该把这个 blocked 决策稳定回显给 Node v269，而不是提前进入真实 resolver 或 managed-audit connection。

本版同时执行计划里的优化项：新增逻辑继续拆到 records / support / builder，`OpsEvidenceService` 只增加版本、profile、endpoint 等常量；没有把大量字段重新堆回主 service。
