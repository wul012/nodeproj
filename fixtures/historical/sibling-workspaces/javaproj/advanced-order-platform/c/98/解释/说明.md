# Java v98 说明：manual sandbox dry-run command package echo receipt

## 计划依据

本轮依据最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v242-post-historical-evidence-fallback-roadmap.md
```

计划要求 Java v98 做：

```text
manual sandbox dry-run command echo receipt
只读回显 Node v241 command package 的 commandCount、credential handle、schema rehearsal id、rollback path、timeout / abort marker
不写 ledger
不执行 SQL
不打开 managed audit connection
```

## 合理性判断

合理。Node v241 已经形成 disabled-by-default dry-run command package，Node v244 需要在继续前只读核对 Java 和 mini-kv 的上游回执。Java 侧适合新增一个明确 receipt，而不是打开真实连接或把 Node command package 当执行许可。

本版同时处理一个小优化：新 builder 内部不直接堆 `true/false` 位置参数，而是用内部 flags record 表达 shape、field echo 和 execution boundary。

