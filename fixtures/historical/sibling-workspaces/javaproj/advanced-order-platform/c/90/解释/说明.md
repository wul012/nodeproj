# v90 运行说明

## 1. 版本目标

Java v90 按 Node `v231-post-preflight-verification-roadmap.md` 继续做低风险质量收敛：把 release approval rehearsal 的上下文归一化和缺失 warning 记录统一到 `ContextHeaderField`，减少 response builder 里的本地 helper。

本版不新增响应字段，不修改 schema version，不改变 JSON contract，不新增 approval 行为，不写 ledger，不执行 SQL。

## 2. 改动范围

新增/调整：

```text
src/main/java/com/codexdemo/orderplatform/ops/ContextHeaderField.java
src/main/java/com/codexdemo/orderplatform/ops/ReleaseApprovalRehearsalResponseBuilder.java
src/main/java/com/codexdemo/orderplatform/ops/ReleaseApprovalRehearsalHintBuilder.java
src/main/java/com/codexdemo/orderplatform/ops/ReleaseApprovalRehearsalHandoffHintBuilder.java
```

## 3. 这版收敛了什么

`ContextHeaderField` 现在同时负责：

```text
trim / blank-to-null
header value / placeholder 选择
missing warning 追加
```

这样 request context、operator window、CI evidence、artifact retention、live readiness、audit persistence handoff、approval record handoff 都复用同一套上下文入口。

## 4. 边界不变

- Java 不新增 approval decision
- Java 不写 approval ledger
- Java 不持久化 approval record
- Java 不写 managed audit store
- Java 不执行 SQL
- Java 不触发 deployment / rollback / restore
- Java 不读取 credential value
- Java 不打开 managed audit connection
