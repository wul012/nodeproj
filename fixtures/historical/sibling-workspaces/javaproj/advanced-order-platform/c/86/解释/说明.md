# v86 运行说明

## 1. 版本目标

Java v86 继续做 release approval rehearsal 的 contract-preserving refactor，重点不是新增 receipt，而是把 builder 内部残留的裸布尔参数收口成语义 record / flags helper，避免在构造链里继续散落 `true/false`。

## 2. 拆分结果

- `ReleaseApprovalRehearsalResponseBuilder`：把 `ExecutionBoundaries` 的只读状态封成内部 `ExecutionBoundaryFlags`。
- `ReleaseApprovalRehearsalHintBuilder`：把 request context、operator window、CI evidence、artifact retention、live readiness 的布尔组改成内部 flags record。
- `ReleaseApprovalRehearsalHandoffHintBuilder`：把 audit persistence、approval record handoff、approval handoff verification marker 的布尔组改成内部 flags record。
- `ReleaseApprovalRehearsalFailureTaxonomyBuilder`：保持不变。

## 3. 代码规模

修改后：

```text
OpsEvidenceService.java: 1443
ReleaseApprovalRehearsalResponseBuilder.java: 398
ReleaseApprovalRehearsalHintBuilder.java: 677
ReleaseApprovalRehearsalHandoffHintBuilder.java: 574
ReleaseApprovalRehearsalFailureTaxonomyBuilder.java: 60
```

主文件仍停在 1443 行；这轮 builder 变长，是因为把原来散在调用处的裸布尔收进了内部语义对象，换的是可读性，不是契约。

## 4. 不变项

- release approval rehearsal API 路径
- response record shape
- schema version
- warning digest 输入顺序
- no-ledger-write proof
- Node verification actions
- all read-only / no-write / no-connection / no-SQL / no-credential flags
