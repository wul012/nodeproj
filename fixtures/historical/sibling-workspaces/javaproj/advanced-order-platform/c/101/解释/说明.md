# Java v101 说明：Dependabot/security maintenance

## 计划依据

本轮依据最新有效全局计划：

```text
D:\nodeproj\orderops-node\docs\plans\v245-post-sandbox-precheck-roadmap.md
```

计划要求 Java v101 做：

```text
Dependabot/security maintenance
覆盖 Maven 与 GitHub Actions
可在 Java v100 CI 基线上运行
不改 release approval 业务语义
```

## 合理性判断

合理。Java v100 已补上 GitHub Actions Maven CI 基线，下一步补 Dependabot 能让 Maven 依赖和 GitHub Actions action 版本自动收到安全维护提醒，并让后续依赖升级进入 PR + CI 审查路径。

本版只新增依赖维护配置，不升级任何依赖版本，不修改 Java 业务代码，不扩大 managed-audit 能力。

## 本版改动

新增：

```text
D:\javaproj\.github\dependabot.yml
```

配置覆盖：

```text
Maven: /advanced-order-platform
GitHub Actions: /
```

维护策略：

```text
weekly schedule
Asia/Shanghai timezone
minor/patch grouped PR
semver-major ignored for automatic version updates
open pull request limit: 5 per ecosystem
```

## 边界

本版没有做：

```text
pom.xml dependency upgrade
GitHub Actions action upgrade
release approval response 字段变更
managed audit sandbox connection
credential value 读取
approval ledger 写入
schema migration SQL
Java / mini-kv / external audit service auto-start
```

## 验证记录

本版执行：

```text
mvn -q -DskipTests compile
git diff --check
```

Dependabot 配置依据 GitHub 官方配置项说明：`package-ecosystem`、`directory`、`schedule`、`groups.update-types` 与 `ignore.update-types` 均为官方支持项。

## 清理记录

验证产生的 `target/` 会在最终收口前删除；本版不保留临时产物。

## 一句话总结

v101 给 Java 仓库补上 Maven + GitHub Actions Dependabot 安全维护入口，让依赖更新走 PR + CI 审查路径；业务契约和 managed-audit 边界不变。
