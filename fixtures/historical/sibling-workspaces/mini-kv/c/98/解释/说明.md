# mini-kv v98 命令归档说明

本轮版本范围：`CommandProcessor::execute_with_wal`。v98 只把 `SET`、`SETNXEX`、`DEL`、`EXPIRE` 的 WAL / no-WAL 重复分支收敛到一个内部 helper，保持 WAL append-before-mutation 顺序、no-op miss 返回、写命令响应、auto compact 触发点、snapshot/restore/admin 行为和所有只读 evidence 契约不变。

本轮开始前只读查看了 Node 计划目录 `D:\nodeproj\orderops-node\docs\plans`，读取 `README.md` 和最新计划 `v231-post-preflight-verification-roadmap.md`。计划要求 mini-kv 继续做 C++ 侧小步、可验证、边界不扩张的质量重构；v98 的 helper 收敛与该方向一致。

## 04-smoke.png

命令：

```powershell
cmake-build-v98\minikv_server.exe 6498 127.0.0.1
cmake-build-v98\minikv_client.exe 127.0.0.1 6498 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端执行：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH

QUIT
```

结果：真实 TCP smoke 成功。`SMOKEJSON`、`INFOJSON`、`STORAGEJSON` 返回 `0.98.0` 只读 evidence；`HEALTH` 返回 OK；`GET restore:real-read-token` 返回 `(nil)`，证明本轮 smoke 没有执行 `SETNXEX` 或产生 restore token。`SMOKEJSON` 继续报告 `read_only=true`、`execution_allowed=false`、`restore_execution_allowed=false`、`order_authoritative=false`、`write_commands_executed=false`、`runtime_write_observed=false`，并保留 Node v233 当前消费提示以及历史 Node v231 no-start guard receipt 链路。

后台进程：本轮 smoke 启动的 `minikv_server.exe` PID 为 `25272`，验证结束后已停止，最终没有保留 mini-kv 后台进程。
