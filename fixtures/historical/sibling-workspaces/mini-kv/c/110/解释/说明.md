# mini-kv v110 运行归档说明

本目录记录 v110 `Dependabot GitHub Actions maintenance` 的关键验证输出。v110 只新增 GitHub Actions 依赖维护配置和 CTest 守卫，不改 WAL/snapshot/restore 核心，不新增 runtime 命令，不启动 Node/Java/mini-kv 服务，不配置 npm/Maven/C++ 包管理生态。

## 图片说明

- `01-cmake-configure.png`：执行 `cmake -S . -B cmake-build-v110 -G Ninja`，确认 CMake 配置和生成成功。
- `02-cmake-build.png`：执行 `cmake --build cmake-build-v110 --parallel 2`，确认库、CLI/server/client、benchmark 和测试程序构建成功。
- `03-ctest.png`：执行 `ctest --test-dir cmake-build-v110 --output-on-failure`，确认全量 CTest 通过，并包含新增 `dependabot_config_tests`。
- `04-dependabot-config-smoke.png`：读取 `.github/dependabot.yml` 和 `.github/workflows/ci.yml`，确认 Dependabot 只覆盖 `github-actions`，周更，Asia/Shanghai 时区，minor/patch 分组，忽略 `version-update:semver-major`，没有配置 npm/Maven/pip/cargo/Docker ecosystem，CI 仍是 CMake/CTest 基线。

## 进程

本版没有启动 `minikv_server.exe` 或其他后台服务。验证只运行 CMake/CTest 和短生命周期配置检查进程，结束后没有保留后台进程。
