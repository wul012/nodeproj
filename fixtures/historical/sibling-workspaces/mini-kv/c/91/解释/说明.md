# mini-kv v91 运行归档说明

本轮归档对应 Node v225 managed audit sandbox adapter dry-run package 的 mini-kv 只读前置守卫。归档重点是证明 v91 可以构建、测试、真实 TCP 读取，并且 `SMOKEJSON` / `INFOJSON` 只暴露沙箱适配器不参与证据，不执行写入、restore、LOAD、COMPACT、SETNXEX、schema migration 或 managed audit state 写入。

## 01-cmake-configure.png

命令：

```powershell
D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe -S . -B cmake-build-v91 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug
```

结果：配置成功，CMake 生成 `cmake-build-v91` 构建文件。

意义：确认 v91 的 CMake 版本号、`runtime_evidence.cpp` 编译单元和测试目标可以被当前工具链识别。

## 02-cmake-build.png

命令：

```powershell
D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe --build cmake-build-v91 --parallel 2
```

结果：所有可执行文件、库和测试目标构建成功。

意义：确认新增 `runtime_evidence` helper、`command.cpp` 的沙箱收据格式化逻辑、夹具断言测试都能通过真实 C++ 编译链接。

## 03-ctest.png

命令：

```powershell
D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\ctest.exe --test-dir cmake-build-v91 --output-on-failure
```

结果：31/31 测试通过。

意义：确认 INFOJSON、SMOKEJSON、release manifest、runtime smoke fixture、历史 receipt 链、v91 sandbox adapter non-participation receipt，以及 WAL/snapshot/restore 非执行边界都被测试保护。

## 04-real-tcp-smoke-client.png

命令：

```powershell
cmake-build-v91\minikv_server.exe 6429 127.0.0.1
cmake-build-v91\minikv_client.exe 127.0.0.1 6429 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端发送：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH
GET restore:real-read-token
QUIT
```

结果：真实 TCP 读取返回 `version="0.91.0"`，包含 `managed_audit_sandbox_adapter_non_participation_receipt`，其 `receipt_digest="fnv1a64:fd925623ea87eac4"`，并返回 `GET restore:real-read-token` 为 `(nil)`。

意义：证明 mini-kv 真实服务端在只读窗口下可被 Node 侧读取；读取过程没有写入恢复 token，没有执行 restore/load/compact，也没有把 mini-kv 变成 sandbox audit storage backend。

## 05-real-tcp-smoke-summary.png

内容：对真实 TCP smoke 的关键字段做摘要校验。

结果：确认 `0.91.0`、沙箱收据 digest、`sandbox_adapter_storage_backend=false`、`credential_value_read_allowed=false`、`sandbox_managed_audit_state_write_allowed=false`、`(nil)` 和 `HEALTH OK` 都出现；验证结束后服务端进程已停止。
