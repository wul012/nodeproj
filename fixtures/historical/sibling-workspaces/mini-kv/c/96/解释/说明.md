# mini-kv v96 命令归档说明

本轮版本范围：`sandbox connection receipt echo marker`。mini-kv 只给 Node v229 提供 Node v228 manual sandbox connection operator packet 字段名的只读回声证据，不执行 sandbox connection，不读取 credential value，不运行 schema rehearsal，不写 managed audit state，不执行 LOAD/COMPACT/SETNXEX/RESTORE。

D:\CLION\CLion 2025.1.1\bin\cmake\win\x64\bin\cmake.exe -S . -B cmake-build-v96 -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Debug

## 04-smoke.png

命令：

```powershell
cmake-build-v96\minikv_server.exe 6496 127.0.0.1
cmake-build-v96\minikv_client.exe 127.0.0.1 6496 5000 --connect-retries 10 --retry-delay-ms 100
```

客户端执行：

```text
SMOKEJSON
INFOJSON
STORAGEJSON
HEALTH
GET restore:real-read-token
QUIT
```

结果：真实 TCP smoke 成功。`SMOKEJSON` 和 `INFOJSON` 返回 v96 read-only evidence，包含 sandbox connection receipt echo marker；`STORAGEJSON` 仍为 read-only storage evidence；`HEALTH` 正常；`GET restore:real-read-token` 返回 `(nil)`，证明本轮没有执行 `SETNXEX` 或产生恢复 token。

后台进程：本轮 smoke 启动的 `minikv_server.exe` PID 为 `17148`，验证结束后已停止。
