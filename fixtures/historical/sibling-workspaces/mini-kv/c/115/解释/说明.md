# mini-kv v115 运行归档说明

## 版本目标

本版新增 disabled credential resolver precheck 非参与回执，供 Node v263 在 Node v262 disabled resolver precheck 之后只读验证。mini-kv 只回显上游字段形状和边界，不实例化 resolver client 或 secret provider，不读取 credential value，不解析 raw endpoint URL，不发送外部请求，不 auto-start，不写 storage，不执行 schema migration、LOAD、COMPACT、RESTORE 或 SETNXEX，也不成为 managed audit storage。

本版还把 credential resolver receipt 实现拆到 `src/runtime_credential_resolver_receipts.cpp`，避免 `src/runtime_sandbox_receipts.cpp` 继续膨胀；真实 TCP smoke 发现 `SMOKEJSON` 已超过旧客户端 64 KiB 响应行上限，因此同步新增 `include/minikv/client_limits.hpp`，把 bundled client 响应行上限提升到 512 KiB。

4. `04-full-ctest.png`
   - 命令：`ctest --test-dir cmake-build-v115 --output-on-failure`
   - 结果：44/44 通过，证明本版没有破坏 store、WAL、snapshot、TCP/RESP、fixture、restore evidence 和历史 receipt 测试。

5. `05-real-tcp-smoke.png`
   - 命令：启动 `minikv_server 6518 127.0.0.1`，再用 `minikv_client 127.0.0.1 6518 5000 --connect-retries 10 --retry-delay-ms 100` 执行 `SMOKEJSON`、`INFOJSON`、`STORAGEJSON`、`HEALTH`、`GET restore:real-read-token`、`QUIT`。
   - 结果：client 退出码为 0，真实 TCP 输出包含 `disabled_credential_resolver_precheck_non_participation_receipt`、`read_only=true`、`execution_allowed=false`，`HEALTH` 返回 OK，`GET restore:real-read-token` 仍为 `(nil)`；验证结束后 server 进程已停止。
