# v168 运行调试说明：production environment preflight checklist

## 本版目标

v168 按最新全局计划推进 Node 自己负责的版本：

```text
Node v168：production environment preflight checklist
```

推进前已只读核对完成：

```text
Java v59：production secret source contract 已完成并推送
mini-kv v68：artifact digest compatibility matrix 已完成并推送
Node v167：rollback execution preflight contract 已完成并推送
```

本版只消费三边证据，生成生产环境前置 checklist，不启动、不停止、不构建、不测试、不修改 Java / mini-kv。

## 安全边界

本版 HTTP smoke 和截图只启动 Node 本地服务，验证后停止。

运行环境保持：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

本版没有启动、停止、构建、测试或修改 Java / mini-kv。

Node v168 明确不允许：

```text
Read production secret values from Node
Infer secret rotation state from Node
Modify Java runtime configuration from Node
Connect to a production database
Execute mini-kv LOAD / COMPACT / SETNXEX
Treat mini-kv digest matrix as Java order authority
Treat this checklist as production environment approval
Bypass Node v167 rollback execution preflight contract
```

## 一句话结论

v168 把 Java v59 的 secret source contract、mini-kv v68 的 artifact digest matrix、Node v167 的 rollback execution preflight contract 合成生产环境前置 checklist，让生产化缺口更可审查，但仍不读取 secret、不连接生产库、不执行 restore、不授权生产操作。
