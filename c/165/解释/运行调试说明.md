# v165 运行调试说明：cross-project release bundle gate

## 本版目标

v165 按最新全局计划推进 Node 自己负责的版本：

```text
Node v165：cross-project release bundle gate
```

前置条件已经只读核对完成：

```text
Java v56：release bundle manifest 已完成并推送
mini-kv v65：runtime artifact bundle manifest 已完成并推送
```

本版只消费两边已归档的 bundle 证据，不启动、不停止、不构建、不测试、不修改 Java / mini-kv。

## 安全边界

本版 HTTP smoke 和截图只启动 Node 本地服务，验证后已停止。

运行环境保持：

```text
UPSTREAM_PROBES_ENABLED=false
UPSTREAM_ACTIONS_ENABLED=false
```

本版没有启动、停止、构建、测试或修改 Java / mini-kv。

Node v165 明确不允许：

```text
Run Maven from Node
Run CMake or CTest from Node
Trigger Java release or rollback from Node
Execute mini-kv LOAD / COMPACT / SETNXEX
Connect production database or production secrets
Treat gate as production release approval
```

## 一句话结论

v165 把 Java v56 与 mini-kv v65 的 release bundle 证据合成了一个只读跨项目 gate，让 Node 更接近生产发布前的统一证据入口，但仍不授权真实发布、回滚或上游写操作。
