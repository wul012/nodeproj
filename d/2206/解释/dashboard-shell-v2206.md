# v2206 Dashboard 页面壳质量收口

v2205 的职责是保持字节不变地拆分大文件，因此浏览器验收发现的三个旧问题没有被偷偷混入重构：`dashboardMarkup` 自带 `</head><body>`，与 `dashboardHtml` 的完整文档壳叠加；浏览器因没有图标声明而额外请求 `/favicon.ico` 并得到 404；长英文审计按钮在 390px 视口内拒绝收缩，宽度超过 302px 内容列。v2206 把这些问题作为独立、可审查的页面质量版本处理，没有增加功能、路由或治理链。

本版先扩充既有 `dashboard.test.ts`。实现前共 5 个用例，其中 3 个按预期失败，分别证明文档壳重复、内嵌 favicon 缺失、按钮缺少收缩与换行约束。随后由 `dashboardHtml` 单独拥有 doctype、html、head、body 和 favicon 元数据；`dashboardMarkup` 只保留 body 内部内容；CSS 将行内按钮从禁止收缩改为允许收缩，并同时加入 `min-width: 0`、`max-width: 100%`、`overflow-wrap: anywhere` 和正常换行。这里没有采用隐藏溢出或省略文字，因为运维按钮的完整名称属于操作语义，必须可读。

结构测试转绿后，旧 v2205 字节 oracle 只因预期中的页面壳变化失败。新基线明确命名为 v2206 结果：markup 从 38137 降为 38122 字节，SHA-256 为 `61219ccf...760c6`；完整 HTML 因加入内嵌 SVG 和 CSS 从 95724 增为 96100 字节，SHA-256 为 `20cd81e4...c95a`。34 条既有状态路由的顺序与处理器仍由同一 parity 测试覆盖，没有修改 API、JSON、Markdown、fixture、跨项目证据或执行权限。

本地服务在 31206 端口完成 6/6 HTTP smoke：health、metrics、runtime config、real-read JSON、同一路由 Markdown 和 Dashboard 均返回 200，Dashboard 响应为 96100 字节。Playwright 在 1440x900 和 390x844 两个视口验证页面；控制台 0 错误，网络记录只有根页面和 13 个业务 API 请求，没有 favicon 请求。移动端按钮区宽 302px，最长按钮宽度也为 302px，并通过 60px 高度自然换行，不再越界。桌面与移动截图分别保存在本版 `图片` 目录。浏览器和受托管服务已经关闭，31206 端口监听数为 0。

维护性 census 保持 `89 / 121 / 238 / 2`，本版新增路由数和服务数均为 0，未放宽任何 ratchet。Java 与 mini-kv 可以继续各自的未提交维护切片；Node 不读取它们的脏工作树，也不触发 live capstone。
