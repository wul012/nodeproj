本次版本目标：

```text
把订单从“支付后结束”推进到真实订单系统常见的履约闭环：

CREATED -> PAID -> SHIPPED -> COMPLETED

CREATED -> CANCELLED

```

# 1. 本次新增能力

第五版新增订单发货和完成能力：

```text
POST /api/v1/orders/{orderId}/ship
 -> 只有 PAID 订单可以发货
 -> 成功后状态变成 SHIPPED
 -> 写入 OrderShipped Outbox 事件

POST /api/v1/orders/{orderId}/complete
 -> 只有 SHIPPED 订单可以完成
 -> 成功后状态变成 COMPLETED
 -> 写入 OrderCompleted Outbox 事件
```

新增状态：

```java
public enum OrderStatus {
    CREATED,
    PAID,
    SHIPPED,
    COMPLETED,
    CANCELLED
}
```

核心领域方法：

```java
public boolean ship() {
    if (status == OrderStatus.SHIPPED) {
        return false;
    }
    if (status != OrderStatus.PAID) {
        throw new BusinessException(HttpStatus.CONFLICT, "ORDER_STATUS_INVALID",
                "Only PAID orders can be shipped");
    }
    status = OrderStatus.SHIPPED;
    shippedAt = Instant.now();
    return true;
}
```

```java
public boolean complete() {
    if (status == OrderStatus.COMPLETED) {
        return false;
    }
    if (status != OrderStatus.SHIPPED) {
        throw new BusinessException(HttpStatus.CONFLICT, "ORDER_STATUS_INVALID",
                "Only SHIPPED orders can be completed");
    }
    status = OrderStatus.COMPLETED;
    completedAt = Instant.now();
    return true;
}
```

这两个方法返回 `boolean`，用于区分：

```text
true
 -> 本次调用真的改变了状态，需要写 Outbox 事件

false
 -> 重复调用，保持幂等，不重复写事件
```

# 7. 当前结论

第五版完成后，项目订单状态机已经从：

```text
CREATED -> PAID
CREATED -> CANCELLED
```

升级为：

```text
CREATED -> PAID -> SHIPPED -> COMPLETED
CREATED -> CANCELLED
CREATED -> EXPIRED(CANCELLED)
```

并且每个关键状态变化都能进入 Outbox：

```text
OrderCreated
OrderPaid
OrderCancelled
OrderExpired
OrderShipped
OrderCompleted
```

一句话总结：第五版把项目从“订单交易闭环”继续推进到“交易 + 履约闭环”，更接近真实电商订单系统。
